# pace-pulse — 节奏心跳检查

系统级 Skill，由 Claude 自动调用。用户不可直接调用。

## 核心特性

- **17 种节奏信号**：覆盖效率、质量、计划、风险、运维、治理六个维度的异常检测
- **非打断式**：0-2 行建议附加在 checkpoint 之后，不中断工作流
- **可操作建议**：每条建议包含可直接执行的命令（如 `/pace-guard report`）
- **会话全生命周期覆盖**：会话开始（分层摘要）+ 推进模式（周期性脉搏）+ 会话结束（节奏总结）

## 关键增强（v1.6）

### 动态信号优先级
信号权重根据迭代阶段动态调整：早期优先关注 CR 停滞和需求冲突；冲刺期优先关注时间压力和审查积压。详见 `pulse-procedures.md` "动态信号权重"章节。

### 信号关联分析
当同组 2+ 信号同时触发时，输出综合诊断而非独立建议。分组：效率、质量、计划、风险。详见 `pulse-procedures.md` "信号分组与关联诊断"章节。

### 会话结束节奏摘要
输出 1-2 行会话级总结，包含变更完成统计、显著节奏信号、迭代进度变化。前缀 `📊`。

### 严重度升级
同一信号在后续检测中严重度翻倍时可再次提醒（如审查积压从 3 涨到 6+），以 `⚠️` 前缀标注。

### 正向反馈
在关键节点（CR merged、迭代过半）无异常时输出 `✓ 节奏健康`（每会话最多 1 次，不计入配额）。

### 写入量提醒协调
pulse-counter Hook 检查 pace-pulse 最近执行时间（< 5 分钟内），避免双重提醒。
