# devpace 需求

> **职责**：定义用户场景、功能需求和非功能需求。需求变更时更新本文件。

## 用户场景

### S1：首次初始化

**角色**：开发者
**场景**：开始一个新项目，想用 devpace 管理研发流程
**期望**：运行一个命令，通过对话式引导完成初始化，得到完整的项目状态目录

**验收标准**：
- [ ] `/pace-init` 通过对话收集项目信息
- [ ] 自动生成 .devpace/ 目录和所有必需文件
- [ ] 生成的价值功能树结构合理

### S2：日常推进

**角色**：开发者
**场景**：新开一个 Claude Code 会话，继续昨天的工作
**期望**：Claude 自动知道做到哪了，直接继续，不需要我重新解释

**验收标准**：
- [ ] Claude 会话开始自动读取 state.md
- [ ] 用 1 句自然语言报告当前状态
- [ ] 用户说"继续"就能从上次断点恢复

### S3：查看进度

**角色**：开发者/项目经理
**场景**：想快速了解项目整体进展
**期望**：不同粒度的进度报告，从 1 句话概览到完整功能树

**验收标准**：
- [ ] `/pace-status` 默认 1-3 行概览
- [ ] `/pace-status detail` 展开到产品功能级别
- [ ] `/pace-status tree` 展示完整价值功能树

### S4：需求插入

**角色**：开发者
**场景**：做到一半，发现需要加一个新功能
**期望**：告诉 Claude 新需求，它自动分析影响、调整计划

**验收标准**：
- [ ] Claude 自动识别"新需求"意图
- [ ] 执行影响分析：影响哪些 PF/CR，迭代容量够不够
- [ ] 提出调整方案，等待确认
- [ ] 确认后自动更新所有相关文件（功能树、迭代计划、state）

### S5：需求暂停/砍掉

**角色**：开发者
**场景**：某个功能暂时不做了，或者优先级降低
**期望**：已有工作成果不丢失，依赖关系自动调整

**验收标准**：
- [ ] CR 标记为 paused，工作成果保留（分支、代码、质量检查进度）
- [ ] 依赖此 CR 的其他 CR 自动解除阻塞或标记受影响
- [ ] 功能树中用 ⏸️ 标记暂停项
- [ ] 恢复时从暂停前状态继续

### S6：优先级重排

**角色**：开发者
**场景**：某个功能需要提前做
**期望**：调整优先级后迭代计划自动重排

**验收标准**：
- [ ] state.md 的"下一步"更新为新的最高优先项
- [ ] 迭代计划反映调整后的顺序
- [ ] 变更记录保留调整原因

### S7：需求修改

**角色**：开发者
**场景**：已有功能的范围或要求变了
**期望**：Claude 评估变更对已有工作的影响

**验收标准**：
- [ ] 识别已完成工作中需要返工的部分
- [ ] 质量检查重置为未检查（需要重新验证）
- [ ] CR 事件记录变更原因

### S8：Code Review

**角色**：开发者
**场景**：Claude 做完一个 CR，等待人类审核
**期望**：清晰的变更摘要，一键审批或打回

**验收标准**：
- [ ] 自动生成变更摘要（改了什么、为什么、影响）
- [ ] 审批后自动合并和状态更新
- [ ] 打回后回到 developing 并记录原因

### S9：迭代回顾

**角色**：开发者/项目经理
**场景**：一个阶段结束，复盘进展
**期望**：基于数据的回顾报告，包括变更造成的偏差分析

**验收标准**：
- [ ] 自动收集质量和交付数据
- [ ] 生成回顾报告（交付/质量/价值/变更影响）
- [ ] 更新度量仪表盘

## 功能需求

### F1：核心工作流

| ID | 功能 | 对应场景 | 优先级 |
|----|------|---------|:------:|
| F1.1 | 项目初始化（/pace-init） | S1 | P0 |
| F1.2 | 自动上下文恢复 | S2 | P0 |
| F1.3 | 变更请求状态机 | S2 | P0 |
| F1.4 | 质量检查自动检查 | S2 | P0 |
| F1.5 | 状态查看（/pace-status） | S3 | P0 |
| F1.6 | 显式推进（/pace-advance） | S2 | P0 |
| F1.7 | Review 摘要（/pace-review） | S8 | P0 |

### F2：需求变更管理

| ID | 功能 | 对应场景 | 优先级 |
|----|------|---------|:------:|
| F2.1 | 变更意图自动识别 | S4-S7 | P0 |
| F2.2 | 影响分析 | S4-S7 | P0 |
| F2.3 | 调整方案生成 | S4-S7 | P0 |
| F2.4 | CR 暂停/恢复 | S5 | P0 |
| F2.5 | 优先级重排 | S6 | P0 |
| F2.6 | 需求修改 + 质量检查重置 | S7 | P1 |
| F2.7 | 显式变更管理（/pace-change） | S4-S7 | P1 |

### F3：度量与回顾

| ID | 功能 | 对应场景 | 优先级 |
|----|------|---------|:------:|
| F3.1 | 迭代回顾（/pace-retro） | S9 | P1 |
| F3.2 | 变更偏差分析 | S9 | P1 |
| F3.3 | 度量仪表盘 | S9 | P1 |

## 非功能需求

| ID | 需求 | 标准 |
|----|------|------|
| NF1 | 零摩擦交互 | 用户全程自然语言，不需要学术语或操作文件 |
| NF2 | 会话启动速度 | state.md ≤ 15 行，加载不增加明显延迟 |
| NF3 | 中断容错 | 任意时刻中断，下次无缝恢复 |
| NF4 | 可移植性 | 新项目 /pace-init 到首个 CR merged < 1 小时 |
| NF5 | 不侵入项目 | .devpace/ 可 .gitignore，不影响项目本身 |
