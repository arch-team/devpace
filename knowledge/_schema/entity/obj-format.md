# 业务目标文件格式契约

> **职责**：定义 OBJ 独立文件 `objectives/OBJ-xxx.md` 的结构。/pace-init 或 /pace-biz 创建时遵循此格式。

## §0 速查卡片

```
文件名：OBJ-xxx.md（xxx 为自增数字，三位补零）
目录：.devpace/objectives/
创建时机：/pace-init（从项目描述推断）或 /pace-biz（用户讨论业务目标时）
始终独立文件：OBJ 不内联在 project.md 中，始终有独立文件
内容：类型 + 时间维度 + 状态 + 北极星贡献 + 双维度 MoS + 关联专题/需求
类型：business | product | tech | growth | efficiency | compliance（6 类）
状态：活跃 | 已达成 | 已废弃（3 态简化）
MoS 格式：双维度——客户价值 + 企业价值（渗透到 OBJ/Epic/BR 三层）
project.md 保留：§2 业务目标 section 改为摘要索引表（链接到独立文件）
向后兼容：无 objectives/ 目录时 project.md §2 保持原有内联 ### OBJ-N 格式（现有行为不变）
容错：OBJ 文件丢失时从 project.md 索引表 + Epic 文件 OBJ 关联字段重建
```

## 文件结构

```markdown
# OBJ-xxx：[目标描述]

- **类型**：[business | product | tech | growth | efficiency | compliance]
- **时间维度**：[短期 | 中期 | 长期]
- **状态**：[活跃 | 已达成 | 已废弃]
- **北极星贡献**：[这个 OBJ 如何贡献到北极星指标]

## 成效指标（MoS）

**客户价值**：
- [ ] [指标 1]（目标：[值]，当前：[值] → 进度 [N]%）

**企业价值**：
- [ ] [指标 2]（目标：[值]，当前：[值] → 进度 [N]%）

## 关联专题

| Epic | 标题 | 关联类型 | 状态 | 完成度 |
|------|------|:--------:|------|:------:|
| EPIC-001 | [专题名] | 主 | 进行中 | 1/3 |
| EPIC-003 | [专题名] | 副 | 规划中 | 0/2 |

## 关联需求（无 Epic 时）

| BR | 标题 | 优先级 | 状态 |
|----|------|:------:|------|
| BR-001 | [需求名] | P0 | 进行中 |
```

> **"关联专题"与"关联需求"互斥**：有 Epic 时仅显示"关联专题"表（BR 挂在 Epic 下）；无 Epic 时仅显示"关联需求"表（BR 直挂 OBJ）。两者不会同时出现在同一个 OBJ 文件中。

## 字段说明

| 字段 | 核心/渐进 | 来源 | 说明 |
|------|:---------:|------|------|
| 标题 | 核心 | 人类提供 | OBJ 编号 + 目标描述 |
| 类型 | 核心 | Claude 推断或人类指定 | 6 类之一，影响 MoS 倾向 |
| 时间维度 | 渐进 | 人类指定或 Claude 推断 | 短期（当前迭代）/中期（2-3 迭代）/长期（季度+） |
| 状态 | 核心 | Claude 自动维护 | 基于 MoS 达成情况和人类确认 |
| 北极星贡献 | 渐进 | 人类定义或 Claude 建议 | 描述 OBJ 与 Vision 北极星指标的关系 |
| MoS 客户价值 | 核心(渐进丰富) | 人类定义或 /pace-biz 引导 | 初始有基本形式，/pace-biz 引导后完善双维度 |
| MoS 企业价值 | 核心(渐进丰富) | 人类定义或 /pace-biz 引导 | 初始有基本形式，/pace-biz 引导后完善双维度 |
| 关联专题 | 核心(自动) | Epic 创建时 Claude 自动关联 | 含关联类型（主/副）、状态、完成度 |
| 关联需求 | 核心(自动) | 无 Epic 时 BR 直挂 OBJ | 向后兼容——无 Epic 时 BR 直接关联 |

## OBJ 类型定义

| 类型 | 含义 | MoS 倾向 | 示例 |
|------|------|---------|------|
| `business` | 业务收入/盈利 | 企业价值：收入、利润率、份额 | "提升订阅收入 30%" |
| `product` | 产品体验/功能 | 客户价值：NPS、完成率、满意度 | "提升核心功能易用性" |
| `tech` | 技术基础设施 | 企业价值：性能、可用性、债务 | "降低 API 延迟到 200ms" |
| `growth` | 用户增长/市场 | 双维度：DAU(客户)、转化率(企业) | "月活用户增长到 10K" |
| `efficiency` | 运营/研发效率 | 企业价值：交付频率、成本、周期 | "缩短发布周期到 1 周" |
| `compliance` | 合规/安全 | 双维度：数据安全(客户)、合规率(企业) | "通过 SOC2 认证" |

## 状态规则

| 状态 | 含义 | 转换条件 |
|------|------|---------|
| 活跃 | 正在追踪的目标 | 创建时默认状态 |
| 已达成 | MoS 全部或大部分达成 | 人类确认 MoS 达成（/pace-retro 时建议） |
| 已废弃 | 目标不再追踪 | 人类显式废弃（战略调整、业务变化） |

**简化设计**：3 态足矣——OBJ 不需要"暂停"状态（暂停是 CR/Epic 级概念，OBJ 级别的"暂停"等价于优先级调整）。

## 渐进阶段

| 阶段 | 触发时机 | 内容 | 典型大小 |
|------|---------|------|:--------:|
| 初始 | /pace-init 推断或 /pace-biz 创建 | 标题 + 类型 + 状态 + 基本 MoS | 8-12 行 |
| 成长 | Epic 关联或北极星讨论后 | + 北极星贡献 + 关联专题表 + 完整双维度 MoS | 15-25 行 |
| 成熟 | 多次 /pace-retro 后 | 全字段完整，MoS 有进度数据 | 20-35 行 |

## 命名规则

- 文件名：`OBJ-001.md`、`OBJ-002.md`...（三位补零自增）
- ID 自增：扫描 `.devpace/objectives/` 中现有 OBJ 文件的最大编号 +1
- 目录不存在时自动创建

## project.md 中的引用格式

**有 objectives/ 时**：§2 业务目标 section 改为摘要索引表：

```markdown
## 业务目标

| OBJ | 目标 | 类型 | 状态 | MoS 进度 |
|-----|------|------|------|---------|
| [OBJ-001](objectives/OBJ-001.md) | [描述] | business | 活跃 | 2/3 |
| [OBJ-002](objectives/OBJ-002.md) | [描述] | product | 活跃 | 1/2 |
```

**无 objectives/ 时**：保持原有内联 `### OBJ-N：...` 格式（向后兼容，现有行为不变）。

### 价值功能树中的 OBJ 引用

**有 objectives/ 时**：

```markdown
[OBJ-001：目标名](objectives/OBJ-001.md)
├── [EPIC-001：专题名](epics/EPIC-001.md)
│   └── BR-001：需求名 ...
```

**无 objectives/ 时**：保持原有纯文本格式 `OBJ-001（目标名）`（向后兼容）。

## 更新时机

| 事件 | 更新内容 |
|------|---------|
| /pace-init | 创建初始 OBJ 文件（从项目描述推断） |
| /pace-biz | 创建新 OBJ 或更新 MoS |
| Epic 创建/状态变更 | 关联专题表 |
| BR 创建（无 Epic 时） | 关联需求表 |
| /pace-retro | MoS 进度更新、状态建议（需人类确认） |
| 战略调整 | 状态变更（需人类确认） |

## 迁移路径

**从 project.md 内联 OBJ 迁移到 objectives/ 独立文件**：

- **触发条件**：Claude 检测到 project.md §2 有内联 `### OBJ-N` 格式但无 `objectives/` 目录
- **步骤**：Claude 建议迁移 → 用户确认 → 创建 objectives/ 目录和 OBJ 文件 → project.md §2 改为索引表
- **需用户确认**：迁移涉及目录结构变更，非静默操作

## 容错

| 异常 | 处理 |
|------|------|
| OBJ 文件丢失 | 从 project.md 索引表 + Epic 文件的 OBJ 关联字段重建 |
| objectives/ 目录不存在 | /pace-init 或 /pace-biz 时自动创建 |
| project.md 链接指向不存在的 OBJ 文件 | 按"OBJ 文件丢失"处理 |
| OBJ 文件与 project.md 索引表状态不一致 | 以 OBJ 文件为准（更详细），同步更新索引表 |
| 关联专题表与实际 Epic 文件不一致 | 以 Epic 文件和 project.md 树视图为准重建 |
| MoS 无维度标签 | 向后兼容——简单 checkbox 列表仍合法，视为未分类 |
| OBJ 部分字段缺失 | 保持已有字段，缺失字段不影响功能——渐进填充 |
