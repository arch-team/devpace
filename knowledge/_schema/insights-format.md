# 经验积累文件格式契约

> **职责**：定义 insights.md 的结构。pace-learn 自动写入和 §12 经验引用时遵循此格式。

## §0 速查卡片

```
insights.md 是项目级经验积累文件
位置：.devpace/metrics/insights.md
写入者：pace-learn（自动）+ §12.5 纠正即学习（用户确认后）
读取者：§12 经验驱动决策（5 个引用时机）
条目类型：模式(pattern) · 防御(defense) · 改进(improvement) · 偏好(preference)
偏好类型优先级 > 模式类型（用户纠正 > 统计规律）
置信度：0.2-0.9 动态范围（初始 0.5 | 验证 +0.1 | 存疑 -0.2 | 上限 0.9 | 下限 0.2）
引用过滤：<0.4 不主动引用 | 0.4-0.7 高匹配时引用 | >0.7 优先引用
最近引用：记录最近一次被 §12 引用的日期
```

## 文件结构

```markdown
# 经验积累

## 模式

### [pattern 标题]

- **类型**：[模式 | 防御 | 改进 | 偏好]
- **来源**：[CR-xxx merged | Gate N 修复 | 用户纠正]
- **标签**：[功能/修复/重构/质量/变更/审批/...]
- **描述**：[1-2 句经验总结]
- **置信度**：[0.2-0.9]
- **最近引用**：[日期 或 （未引用）]
- **验证次数**：[N]（同类场景再次验证时 +1）
- **日期**：[首次记录日期]

---
```

## 条目类型

| 类型 | 语义 | 来源 | 引用优先级 |
|------|------|------|-----------|
| 模式（pattern） | 重复出现的成功做法 | pace-learn 从 merged CR 提取 | 标准 |
| 防御（defense） | 需要预防的风险/问题 | pace-learn 从 Gate 修复/打回提取 | 标准 |
| 改进（improvement） | 流程改进建议 | pace-learn 从 retro 提取 | 标准 |
| **偏好（preference）** | **用户纠正形成的行为偏好** | **§12.5 纠正即学习，用户确认后写入** | **高于模式/防御/改进** |

## 偏好类型条目

偏好类型是用户对 Claude 判断的纠正记录。当 §12 引用时，偏好类型优先级高于其他类型——用户的明确纠正 > 统计规律。

### 偏好条目结构

```markdown
### [偏好标题]

- **类型**：偏好
- **来源**：用户纠正（[纠正行为描述]）
- **标签**：[场景标签]
- **描述**：[用户偏好的行为，以及与 Claude 默认行为的差异]
- **纠正场景**：[触发纠正的原始场景]
- **调整后行为**：[Claude 在后续同类场景中应采取的行为]
- **置信度**：[0.2-0.9]
- **最近引用**：[日期 或 （未引用）]
- **验证次数**：[N]
- **日期**：[记录日期]
```

### 纠正场景示例

| 纠正行为 | Claude 原始判断 | 偏好条目描述 |
|---------|------------------|------------|
| 用户否决简化审批 | 满足简化审批条件 | "涉及 [模块] 的 CR 不适合简化审批，即使满足条件也走完整 review" |
| 用户修改 PF 分解 | Claude 推断了 PF 结构 | "该业务域偏好 [用户的分解方式] 而非 [Claude 的分解方式]" |
| 用户修改影响评估 | Claude 判断为低风险 | "涉及 [模块] 的变更应评估为更高风险" |
| 用户否决 Gate 2 结果 | Claude 判断验收通过 | "该类验收条件需要更严格的判断标准" |

## 标签系统

标签用于 §12 引用时的匹配。标签不预定义，由 pace-learn 从 CR 特征自动生成。

常用标签参考：`功能`、`修复`、`重构`、`质量`、`变更`、`审批`、`性能`、`安全`、`测试`、`文档`

## 置信度规则

置信度反映 pattern 的可靠程度，动态范围 0.2-0.9。

### 基本规则

| 事件 | 置信度变化 | 说明 |
|------|-----------|------|
| 新建 pattern | 初始 0.5 | 首次提取的 pattern |
| 新建偏好 | 初始 0.7 | 用户纠正形成的偏好（更高可信度） |
| 验证（同类成功） | +0.1 | 同类场景再次验证时 |
| 存疑（反例出现） | -0.2 | 出现与 pattern 矛盾的证据 |

### 边界

- **上限**：0.9（永远保留怀疑空间）
- **下限**：0.2（不自动删除，但极低优先级）
- **计算**：加减后 clamp 到 [0.2, 0.9] 范围

### 引用过滤

§12 经验引用时按置信度过滤：

| 置信度范围 | 引用行为 |
|-----------|---------|
| > 0.7 | 优先引用——主动推荐 |
| 0.4 - 0.7 | 条件引用——仅在高度匹配当前场景时引用 |
| < 0.4 | 不主动引用——仅用户主动查询时可见 |

### 最近引用字段

- `最近引用` 记录该 pattern 最近一次被 §12 引用的日期
- 未被引用过的 pattern 标记为 `（未引用）`
- 用于未来的置信度衰减扩展（长期不引用可降低置信度）

## 跨项目导出/导入

### 导出规则

从项目 A 的 insights.md 导出可复用经验：

1. **过滤条件**：置信度 ≥ 0.7 且类型 ≠ 偏好（preference）
   - 偏好类型是项目特定的用户习惯，不可跨项目迁移
   - 低置信度 pattern 未经充分验证，不适合迁移
2. **导出格式**：与 insights.md 条目格式一致，额外添加来源标记：
   - `- **导出源**：[项目名称] ([导出日期])`
3. **导出位置**：输出到用户指定路径（默认 `./insights-export.md`）

### 导入规则

在项目 B 中导入其他项目的经验：

1. **触发方式**：`/pace-init --import-insights <导出文件路径>`
2. **置信度降级**：导入 pattern 的置信度 = 原值 × 0.8（需在新项目中重新验证）
   - 示例：原置信度 0.9 → 导入后 0.72，原置信度 0.7 → 导入后 0.56
   - 降级后仍遵循 [0.2, 0.9] 范围约束
3. **偏好过滤**：导入时再次校验——如果导出文件中包含偏好类型条目，静默跳过
4. **去重**：如果目标 insights.md 已有同标题 pattern，保留置信度较高的版本
5. **来源标记**：导入的条目保留 `导出源` 字段，并追加 `- **导入日期**：[日期]`
6. **验证次数重置**：导入的 pattern 验证次数重置为 0（需在新项目中重新积累）

### 导出文件格式

```markdown
# 经验导出

> 导出自：[项目名称] | 日期：[导出日期] | 条目数：[N]
> 过滤条件：置信度 ≥ 0.7，排除偏好类型

## 模式

### [pattern 标题]

- **类型**：[模式 | 防御 | 改进]
- **来源**：[原始来源]
- **导出源**：[项目名称] ([导出日期])
- **标签**：[标签列表]
- **描述**：[经验总结]
- **置信度**：[原始置信度]
- **验证次数**：[原始验证次数]
- **日期**：[首次记录日期]

---
```

## 更新规则

- **pace-learn 写入**：CR merged / Gate 修复 / 人类打回时自动提取 pattern
- **纠正即学习写入**：用户纠正 + 用户确认后写入偏好条目（见 devpace-rules.md §12.5）
- **验证次数递增**：同类场景再次验证时，已有 pattern 的验证次数 +1，置信度 +0.1（clamp 0.9）
- **存疑记录时**：置信度 -0.2（clamp 0.2）
- **§12 引用时**：更新该 pattern 的"最近引用"日期
- **不自动删除**：pattern 积累后长期保留——置信度 < 0.4 的 pattern 保留但不主动引用
- **目录不存在时**：pace-learn 自动创建 `metrics/` 目录和 `insights.md` 文件
