# 迭代文件格式契约

> **职责**：定义 iterations/current.md（及归档迭代文件）的结构。/pace-init 创建、/pace-retro 和 /pace-change 读写。

## §0 速查卡片

```
文件名：iterations/current.md（当前迭代）或 iterations/iter-N.md（归档）
标题行：# 迭代：[迭代标识]
头部 blockquote：目标 + 周期（起止日期）
必含：产品功能表 + 变更记录表 + 偏差快照 + 回顾
状态标记：⏳ 待开始 · 🔄 进行中 · ✅ 完成 · ⏸️ 暂停 · 🆕 变更新增 · ⏫ 优先级提升
```

## 文件结构

```markdown
# 迭代：[迭代标识]

> 目标：[迭代目标]
> 周期：[起始日期] — [结束日期]

## 产品功能

| PF | 标题 | 优先级 | 状态 | CR 数 | 完成 | Release |
|----|------|:------:|------|:-----:|:----:|---------|
| [PF 编号] | [PF 标题] | — | ⏳ | 0 | 0 | — |

状态标记：⏳ 待开始 · 🔄 进行中 · ✅ 完成 · ⏸️ 暂停 · 🆕 变更新增 · ⏫ 优先级提升

## 变更记录

| 日期 | 类型 | 描述 | 影响 |
|------|------|------|------|

## 偏差快照

- **计划 PF 数**：[数字]
- **实际 PF 数**：_迭代结束时填写_
- **变更次数**：_自动统计变更记录表行数_
- **延后/取消 PF**：_迭代结束时填写_
- **缺陷 CR 数**：_迭代内 type:defect CR 数量_

## 回顾

_迭代结束时由 /pace-retro 填写。_
```

注意：模板文件 `skills/pace-init/templates/iteration.md` 使用双花括号占位符（如 ITERATION_ID），本 schema 使用 `[描述]` 标注语义，两者对应关系见下方字段说明。

## 字段说明

### 头部

| 字段 | 说明 | 来源 |
|------|------|------|
| `ITERATION_ID` | 迭代标识（如 "Iter-1" 或自定义名称） | /pace-init 对话 |
| `ITERATION_GOAL` | 迭代目标（1 句话） | /pace-init 对话 |
| `START_DATE` | 起始日期 YYYY-MM-DD | /pace-init 对话 |
| `END_DATE` | 结束日期 YYYY-MM-DD | /pace-init 对话 |

### 产品功能表

| 列 | 说明 |
|----|------|
| PF | PF 编号 |
| 标题 | 产品功能标题 |
| 优先级 | 可选，`P0`（必须完成）/ `P1`（应完成）/ `P2`（尽量完成）/ `—`（未标注）|
| 状态 | emoji 标记（见状态标记说明） |
| CR 数 | 该 PF 下 CR 总数 |
| 完成 | 已 merged 的 CR 数 |
| Release | 关联的 Release ID（如 REL-001），无关联时填 `—` |

### 变更记录表

| 列 | 说明 |
|----|------|
| 日期 | 变更日期 YYYY-MM-DD |
| 类型 | 插入 / 暂停 / 恢复 / 优先级调整 / 范围修改 |
| 描述 | 变更的自然语言描述 |
| 影响 | 受影响的 PF/CR 和范围 |

由 /pace-change 执行后自动追加行。

### 偏差快照

迭代过程中由 /pace-retro 更新，记录计划与实际的偏差数据。

### 回顾

迭代结束时由 /pace-retro 填写回顾内容。

## 写入规则

- /pace-init：创建初始文件，填充头部和产品功能表
- /pace-plan：创建迭代文件（next）、归档为 iter-N.md（close）、更新产品功能表和变更记录（adjust）
- /pace-change：追加变更记录表行，更新产品功能表状态
- /pace-dev：更新产品功能表的 CR 数和完成数
- /pace-retro：更新偏差快照，填写回顾 section
- /pace-release：更新产品功能表的 Release 列
