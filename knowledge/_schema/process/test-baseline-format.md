# 测试基准线格式契约

> **职责**：定义 `.devpace/rules/test-baseline.md` 的结构。`/pace-test baseline` 生成时遵循此格式。

## §0 速查卡片

```
用途：记录测试执行基准线，供 /pace-retro 度量和趋势分析使用
生成命令：/pace-test baseline
消费者：/pace-retro（回顾度量）| /pace-test baseline（历史对比）
核心结构：当前基准表 + 检查项明细表 + 历史趋势表
指标字段：基准日期 | 检查项总数 | 通过率 | 总执行时间 | 需求覆盖率（可选）
趋势标记：↑ 上升 | ↓ 下降 | → 持平 | ↑ 新增
```

---

## 文件结构

```markdown
# 测试基准线

> 由 /pace-test baseline 生成于 [日期]。

## 当前基准

| 指标 | 值 |
|------|---|
| 基准日期 | [日期] |
| 检查项总数 | N |
| 通过率 | X/N (Y%) |
| 总执行时间 | Z.Zs |
| 需求覆盖率 | M/K (W%)（如有） |

## 检查项明细

| # | 检查项 | 状态 | 耗时 | 与上次对比 |
|---|--------|------|------|-----------|
| 1 | [名称] | ✅ PASS | 2.3s | → 持平 |
| 2 | [名称] | ✅ PASS | 1.1s | ↑ 新增 |

## 历史趋势

| 日期 | 通过率 | 执行时间 | 备注 |
|------|--------|---------|------|
| [日期] | X/N (Y%) | Z.Zs | 初始基准 |
| [日期] | X/N (Y%) | Z.Zs | +2 检查项 |
```

## 字段定义

### 当前基准表

| 字段 | 必填 | 说明 |
|------|------|------|
| 基准日期 | 是 | 基准建立/更新的日期 |
| 检查项总数 | 是 | 本次执行的检查项数量 |
| 通过率 | 是 | 通过数/总数（百分比） |
| 总执行时间 | 是 | 所有检查项的累计执行耗时 |
| 需求覆盖率 | 否 | 仅当 test-strategy.md 存在时填写 |

### 检查项明细表

| 字段 | 必填 | 说明 |
|------|------|------|
| # | 是 | 检查项序号 |
| 检查项 | 是 | 检查项名称 |
| 状态 | 是 | `✅ PASS` / `❌ FAIL` |
| 耗时 | 是 | 执行耗时（秒） |
| 与上次对比 | 是 | `→ 持平` / `↑ 新增` / `↓ 已移除` / 耗时变化描述 |

### 历史趋势表

| 字段 | 必填 | 说明 |
|------|------|------|
| 日期 | 是 | 基准建立/更新日期 |
| 通过率 | 是 | 该次基准的通过率 |
| 执行时间 | 是 | 该次基准的总执行时间 |
| 备注 | 否 | 变化说明（如 "+N 检查项"、"初始基准"） |

## 更新规则

- 每次执行 `/pace-test baseline` 时覆盖"当前基准"和"检查项明细"section
- "历史趋势"表追加新行（不覆盖历史记录）
- 首次建立基准时"与上次对比"列统一填"初始"
- 基准反映实际状态（包括失败项），不仅记录理想状态
