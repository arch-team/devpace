# 产品线文件格式契约

> **职责**：定义 product-line.md 的结构。/pace-init 创建和 Claude 更新时遵循此格式。

## §0 速查卡片

```
product-line.md 是项目的业务全景图
包含：愿景 + 业务目标 + 成效指标 + 实施路径 + 价值追溯树
价值追溯树：OBJ → BR → PF → CR 的完整链路
CR 状态用 emoji：🔄 进行中 | ✅ 完成 | ⏳ 待开始
```

## 必需内容

### 1. 项目标题和 blockquote 定位

```markdown
# [项目名称]

> [一句话项目定位]
```

### 2. 业务目标

```markdown
## 业务目标

**当前目标**：[目标 ID] — [目标描述]

**成效指标（MoS）**：
- [ ] [指标 1]
- [ ] [指标 2]
```

成效指标用 checkbox，达成时勾选。

### 3. 实施路径

```markdown
## 实施路径

| Step | 名称 | 状态 |
|------|------|------|
| 1 | [名称] | 🔄 进行中 |
| 2 | [名称] | ⏳ 待开始 |
```

### 4. 价值追溯树

```markdown
## 价值追溯树

OBJ-xxx（[目标名]）
├── BR-001：[业务需求名]
│   ├── PF-001：[产品功能名] → CR-001 🔄
│   └── PF-002：[产品功能名] → (待创建 CR)
└── BR-002：[业务需求名]
    └── PF-003：[产品功能名] → CR-002 ⏳
```

CR 状态标记：
- `🔄` — 进行中
- `✅` — 已完成（merged）
- `⏳` — 待开始
- `(待创建 CR)` — 产品功能已定义但尚未创建变更请求

## 更新时机

- 价值追溯树在 CR 创建/状态变更时由 Claude 自动更新
- 业务目标和成效指标由人类维护
- 实施路径的 Step 状态在对应阶段完成时更新
