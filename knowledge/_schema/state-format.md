# state.md 格式契约

> **职责**：定义项目状态快照文件必须包含的内容。Claude 在更新 state.md 时遵循此格式。

## §0 速查卡片

```
state.md 是会话启动的唯一必读文件
总行数 ≤ 15 行（按复杂度自适应：简单 5-8 / 中等 10-12 / 复杂 13-15）
必含：目标摘要 + 当前工作 + 下一步
可选：迭代行（有 iterations/ 时）、关键决策（中等+）、风险关注（复杂）、发布状态（有 Release 时）
不包含：完整 CR 列表、度量数据、质量检查详情
最小状态（v0.8.0 init）：仅目标摘要 + 当前工作 + 下一步，无迭代行
```

## 必需内容

### 头部 blockquote（1-2 行）

```markdown
> 目标：[业务目标简述] → 成效指标：[关键 MoS]
> 迭代：[迭代 ID]（[迭代目标]）| 进度：M/N 产品功能已完成
```

迭代行为可选——最小初始化时不含迭代行，首次 `/pace-plan` 后添加。目标行中 MoS 可为空（显示为 `成效指标：（待定义）`）。

### 当前工作（列表）

```markdown
- **进行中**：[自然语言标题] → [当前阶段的简述]
- **待 Review**：[自然语言标题]（如有）
- **阻塞**：[阻塞描述]（如有）
```

无内容时写"（无）"。

### 下一步（1-2 行）

Claude 下次会话应该做什么的建议。人类可以修改此字段来指导 Claude 的下次工作方向。

### 版本标记（文件末尾）

```markdown
<!-- devpace-version: 1.3.0 -->
```

用于版本兼容性检查。Claude 读取 state.md 时校验版本，确保与当前 Plugin 版本匹配。版本不匹配时提示用户但不阻断工作。

合法版本值：语义化版本 `X.Y.Z`，当前最新 `1.4.0`。

### 教学标记（文件末尾，版本标记同行或相邻）

```markdown
<!-- taught: cr,gate1,review,change,tree,merge,accept -->
```

记录已向用户解释过的系统行为，用于渐进教学去重（同一行为终身只教一次）。

合法标记值：`cr`（自动创建 CR）、`gate1`（Gate 1 质量检查）、`review`（等待 review）、`change`（变更意图检测）、`tree`（功能树更新）、`merge`（merged 连锁更新）、`accept`（AI 验收验证）、`discover_retro`（功能发现：/pace-retro）、`discover_change`（功能发现：/pace-change）、`discover_plan`（功能发现：/pace-plan）、`discover_release`（功能发现：/pace-release）、`discover_feedback_report`（功能发现：/pace-feedback report）、`first_merged`（首个 CR merged 回顾）。

- 标记缺失时等同于"全部未教"（向后兼容新老项目）
- Claude 在首次触发对应行为时检查标记，已存在则跳过教学
- 教学完成后立即追加标记值

## 可选内容

### 可选段："关键决策"（中等/复杂项目）

```markdown
- **决策**：[决策 1 摘要] | [决策 2 摘要] | [决策 3 摘要]
```

放在"下一步"之前。仅保留最近 3 个决策，旧的删除。

### 可选段："风险关注"（复杂项目）

```markdown
- **风险**：[风险 1] | [风险 2]
```

放在"关键决策"之后、"下一步"之前。仅保留当前最高优先级的 2-3 项。

### 可选段："发布状态"（有 Release 时）

```markdown
- **发布**：[REL-xxx] [状态]（[包含 CR 数]）
```

放在"当前工作"之后。仅显示当前活跃的 Release（staging 或 deployed 状态），已 closed 的不显示。

示例：
```markdown
- **发布**：REL-001 staging（3 CR 待部署）
```

## 容错规则

- **缺失 section**：缺少"当前工作"或"下一步"时，补充默认值"（无）"并继续
- **格式损坏**：头部 blockquote 缺失时，基于已有内容重建，保留原有信息
- **版本缺失**：无 devpace-version 注释时，视为 0.1.0（初始版本）
- **编码兼容**：人类手动编辑可能引入格式不一致（如多余空行、缩进变化），读取时忽略格式差异，只关注语义内容
- **发布段缺失**：无"发布"段时不报错，仅当存在活跃 Release 时才显示

## 自适应内容策略

state.md 在 ≤15 行限制内按项目复杂度自适应调整内容密度：

| 复杂度 | 判断标准 | 内容策略 | 典型行数 |
|--------|---------|---------|:--------:|
| 简单 | ≤3 CR | 标准格式（目标+当前+下一步） | 5-8 |
| 中等 | 4-10 CR | 增加"关键决策"段（最近 3 个重要决策的 1 句话摘要） | 10-12 |
| 复杂 | >10 CR | 增加"风险关注"段（当前最高风险的 2-3 项） | 13-15 |

### 压缩原则

- 保留"下一步该做什么"（行动导向）→ 恢复效率最高
- 保留"最近做了什么重要决策"（决策上下文）→ 避免重复讨论
- 删除"已完成且无后续影响的历史状态"→ 释放行数空间
- 发布状态仅显示活跃 Release → 不占用已关闭 Release 的行数

## 更新时机

- Claude 在每次会话结束时自动更新
- 人类可以随时手动修改（特别是"下一步"字段）
- 复杂度判断在每次更新时重新评估（CR 数量可能变化）
- Release 状态在 Release 创建/状态变更时同步更新
