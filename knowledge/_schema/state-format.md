# state.md 格式契约

> **职责**：定义项目状态快照文件必须包含的内容。Claude 在更新 state.md 时遵循此格式。

## §0 速查卡片

```
state.md 是会话启动的唯一必读文件
总行数 ≤ 15 行
包含：目标摘要 + 当前工作 + 下一步
不包含：完整 CR 列表、度量数据、门禁详情
```

## 必需内容

### 头部 blockquote（1-2 行）

```markdown
> 目标：[业务目标简述] → 成效指标：[关键 MoS]
> 迭代：[迭代 ID]（[迭代目标]）| 进度：M/N 产品功能已完成
```

### 当前工作（列表）

```markdown
- **进行中**：[自然语言标题] → [当前阶段的简述]
- **待 Review**：[自然语言标题]（如有）
- **阻塞**：[阻塞描述]（如有）
```

无内容时写"（无）"。

### 下一步（1-2 行）

Claude 下次会话应该做什么的建议。人类可以修改此字段来指导 Claude 的下次工作方向。

## 更新时机

- Claude 在每次会话结束时自动更新
- 人类可以随时手动修改（特别是"下一步"字段）
