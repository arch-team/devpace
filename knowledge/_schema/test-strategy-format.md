# 测试策略格式契约

> **职责**：定义 `.devpace/rules/test-strategy.md` 的结构。`/pace-test strategy` 生成时遵循此格式。

## §0 速查卡片

```
用途：将 PF 验收条件系统化地映射到测试策略，并提供实施指导
生成命令：/pace-test strategy
消费者：/pace-test coverage（覆盖率分析+阈值检查）| /pace-test impact（变更影响分析）| /pace-test report（汇总报告）
核心结构：策略总览表 + 覆盖率阈值（可选）+ 每个 PF 的验收条件→测试映射表 + 建议列表 + 测试金字塔 + 测试数据策略 + 实施指导
测试类型：unit | integration | E2E | performance | security | accessibility | manual（需人工验证）
状态标记：✅ 已有 | ❌ 待建 | ⚠️ 部分覆盖 | ⚠️ 不稳定
优先级：P0 核心流程 | P1 重要功能 | P2 辅助功能
```

---

## 字段定义

### 策略总览表

| 字段 | 必填 | 说明 |
|------|------|------|
| PF | 是 | PF 编号，与 project.md 价值功能树一致 |
| 功能名 | 是 | PF 标题 |
| 验收条件数 | 是 | 该 PF 下可测试的验收条件总数 |
| 已覆盖 | 是 | 有对应测试（✅ 已有 或 ⚠️ 部分覆盖）的条件数 |
| 覆盖率 | 是 | 已覆盖 / 验收条件数，百分比（需求覆盖率） |
| 代码覆盖率 | 否 | 行覆盖率百分比（由 /pace-test coverage 采集，辅助信号） |
| 优先级 | 是 | P0（核心流程）/ P1（重要功能）/ P2（辅助功能） |

### 验收条件→测试映射表

| 字段 | 必填 | 说明 |
|------|------|------|
| # | 是 | 验收条件序号（与 PF 内验收标准的顺序一致） |
| 验收条件 | 是 | 验收条件文本（从 project.md 提取） |
| 测试类型 | 是 | 主类型：`unit` / `integration` / `E2E` / `performance` / `security` / `accessibility` / `manual`；可含辅助类型后缀：`unit [+security]`（0-2 个辅助类型） |
| 工具/框架 | 是 | 具体测试框架（pytest / jest / playwright / vitest 等）；manual 类型填 `—` |
| 测试文件/位置 | 否 | 已有测试的文件路径；待建时填 `—` |
| 状态 | 是 | `✅ 已有` / `❌ 待建` / `⚠️ 部分覆盖` |

### 建议列表

每个 PF section 末尾的建议列表，按以下优先级排序：

1. P0 功能的 ❌ 待建 测试（最高优先）
2. P1 功能的 ❌ 待建 测试
3. ⚠️ 部分覆盖的测试补全建议
4. P2 功能的测试建议

### 技术栈信息

| 字段 | 必填 | 说明 |
|------|------|------|
| 语言 | 是 | 项目主要编程语言 |
| 测试框架 | 是 | 项目使用的测试框架 |
| 来源 | 是 | 技术栈信息来源：`.devpace/context.md` 或 `自动检测`（注明检测文件） |

### 覆盖率阈值表（可选）

| 字段 | 必填 | 说明 |
|------|------|------|
| 指标 | 是 | `需求覆盖率` 或 `代码覆盖率` |
| 阈值 | 是 | 百分比目标值 |
| 当前值 | 否 | 由 /pace-test coverage 自动填充 |
| 状态 | 否 | `✅` / `❌` / `—`，由 /pace-test coverage 自动填充 |

此 section 为可选——不添加则不执行阈值检查。用户手动添加后，`/pace-test coverage` 自动执行检查。

### 测试金字塔表

| 字段 | 必填 | 说明 |
|------|------|------|
| 测试类型 | 是 | 去重后的测试类型 |
| 数量 | 是 | 该类型的验收条件数量 |
| 占比 | 是 | 该类型占总验收条件的百分比 |
| 评估 | 是 | 健康度评估（✅ 健康 / ⚠️ 偏高或偏低 / — 无评估标准） |

### 测试数据策略

| 字段 | 必填 | 说明 |
|------|------|------|
| 数据准备 | 是 | 推荐的数据准备方式和工具 |
| 隔离策略 | 是 | 推荐的测试隔离方式 |
| 敏感数据 | 是 | 敏感数据处理策略 |
| 环境重置 | 是 | 测试后环境重置方式 |

仅当策略中存在 integration 或 E2E 类型时生成。

### 实施指导

每个出现的测试类型一个子 section，包含：推荐框架、初始化命令、配置文件、运行命令、推荐模式。仅为策略中实际出现的测试类型生成。

## 测试类型

合法值：`unit` | `integration` | `E2E` | `performance` | `security` | `accessibility` | `manual`

辅助类型（可选后缀，每条 0-2 个）：`[+security]` | `[+performance]` | `[+integration]`，输出格式如 `unit [+security]`。

> 推荐映射规则（验收条件→测试类型的判定逻辑）见 `skills/pace-test/test-procedures-strategy.md` §3。

## 状态标记规则

| 状态 | 条件 | 说明 |
|------|------|------|
| ✅ 已有 | 找到对应测试文件且包含相关测试用例 | 该验收条件已有自动化测试覆盖 |
| ❌ 待建 | 未找到对应测试 | 该验收条件需要新建测试 |
| ⚠️ 部分覆盖 | 有测试但未完全覆盖验收条件的所有方面 | 需要补充测试用例 |
