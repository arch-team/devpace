# 分级输出参考指南

> **职责**：分级输出的示例表和格式规范。Rules §5 保留行为指令，本文件提供按需参考的示例和格式细节。

## §0 速查卡片

```
表面层：推理后缀（系统行为后附 ——[推理] ≤15 字）
中间层：追问展开（用户问"为什么" → 2-5 行推理链）
深入层：/pace-trace（完整决策轨迹 + 溯源标记）
```

## 分级输出场景表

| 场景 | 输出量 | 示例 |
|------|--------|------|
| 完成一个步骤 | 1 句话 | "标注完成，框架占比 75%。" |
| 会话结束 | 3-5 行 | 做了什么 + 质量检查概况 + 下一步 |
| 用户问详情 | 按需展开 | 质量检查清单、进度百分比 |
| `/pace-status detail` | 完整状态 | 全部 PF/CR 状态 |
| `/pace-retro` | 度量仪表盘 | dashboard.md 内容 |

## 推理后缀示例表

| 系统行为 | 无推理（旧） | 有推理（新） |
|---------|------------|------------|
| Gate 通过 | "Gate 1 通过。" | "Gate 1 通过——测试 92%，lint 0 error。" |
| Gate 失败 | "Gate 1 未通过，修复中。" | "Gate 1 未通过——auth 模块 2 个测试失败，修复中。" |
| 节奏提醒 | "有 2 个变更等待 review。" | "有 2 个变更等待 review——权限模块阻塞用户管理。" |
| 简化审批 | "自动合并还是要看看？" | "单文件改动，Gate 一次通过，无漂移。自动合并还是要看看？" |
| 变更影响 | "高风险。" | "高风险——涉及 auth 公共接口，3 个下游模块依赖。" |
| 意图检查 | "明确了范围，开始做。" | "明确了范围（2 文件，标准复杂度），开始做。" |
| 经验引用 | "建议注意 [问题]。" | "根据 [pattern 名] 经验——此类变更容易 [问题]。" |

## 中间层格式规范（追问展开）

**触发词**："为什么""怎么判断的""详细说说""依据是什么""解释一下"

**输出格式**（2-5 行）：

```
判断依据：
  1. [主要因素]——[具体数据/事实]
  2. [次要因素]——[具体数据/事实]
  建议：[下一步行动建议]
```

**示例**：

```
用户："为什么高风险？"
Claude："判断依据：
  1. auth/middleware.ts 是公共接口——被 3 个下游模块直接 import
  2. 涉及 JWT 验证逻辑变更——影响所有受保护路由
  3. 无现有测试覆盖这个代码路径
  建议：先补充集成测试再推进。"
```

## 深入层说明（/pace-trace）

用户通过 `/pace-trace` 主动查询完整决策轨迹。详见 `skills/pace-trace/SKILL.md`。

**输出包含**：
- 读取的所有上下文文件
- 逐项检查结果和规则匹配过程
- 溯源标记信息（source: user/claude）
- 综合判定和 confidence 评估
