# 角色适配通用规则（权威源）

> **职责**：定义角色关注维度和跨 Skill 适配原则。所有消费角色数据的 Skill 引用此文件。修改角色定义时仅需更新此处。

## 通用原则

1. 角色适配仅调整**追问方向、展示维度、措辞侧重**，不改变子命令的核心流程和产出结构
2. Dev 角色使用默认策略（零改变），避免对最常见角色产生额外负担
3. 角色不影响写入格式和数据结构，只影响交互体验
4. 无数据支撑特定角色维度时，回退到 Dev 默认格式

## 角色关注维度表

切换后，同一会话中所有后续输出自动应用新角色视角：

| 角色 | /pace-status 关注 | /pace-retro 关注 | 通用输出风格 |
|------|-------------------|------------------|-------------|
| Biz Owner | MoS 达成率、业务价值交付 | ROI、目标对齐、战略偏差 | 业务术语，关联 OBJ |
| PM | 迭代进度、功能完成度、依赖 | 交付效率、范围变更、资源 | 功能维度，关注节奏 |
| Dev | CR 状态、质量门、技术细节 | 代码质量、技术债、复杂度 | 技术术语，关注实现 |
| Tester | 缺陷分布、测试覆盖、验证状态 | 缺陷逃逸、回归风险、覆盖率 | 质量维度，关注缺陷 |
| Ops | Release 状态、部署健康、MTTR | 部署频率、故障恢复、稳定性 | 运维术语，关注稳定 |

## 角色追问与展示维度

| 角色 | 追问侧重 | 展示侧重 | 措辞风格 |
|------|---------|---------|---------|
| Biz Owner | 商业价值、市场影响、ROI | MoS 达成进度、营收相关指标 | 业务导向 |
| PM | 用户场景、优先级、竞品对比 | PF 完成度、依赖关系 | 产品导向 |
| Dev | （默认，零改变） | CR 状态、技术复杂度 | 技术导向 |
| Tester | 可测试性、验收条件、边界 | 验收标准数量、Gate 2 状态 | 质量导向 |
| Ops | 运维影响、部署需求、基础设施 | Release 关联、运维风险 | 运维导向 |

## 适配范围

各 procedure 中的角色适配段引用本文件的通用维度表，并保留子命令特定的追问示例和展示列。通用维度提供一致性基准，子命令特定细节提供上下文精确性。
