# batch 批量变更规程

> **职责**：batch 子命令的完整流程。自包含，按 SKILL.md 路由表按需加载。依赖 `change-procedures-common.md` + `change-procedures-types.md` 的通用框架。

> OPT-04：多变更场景交互次数 N → 1。

## 触发

- 显式：`/pace-change batch "暂停 A 和 B，把 C 提前"`
- 自然语言识别：用户一次表达多个变更意图（"把 A 和 B 都暂停""暂停 A 同时把 C 优先级提高"）

## 流程

1. **意图解析**：从用户输入中提取所有变更意图，逐条列出确认
   - 格式："检测到 N 个变更意图：① [类型] [描述]  ② [类型] [描述] ... 正确吗？"
2. **合并影响分析**：所有变更共用一次上下文加载，合并执行 Step 1
3. **交叉影响检测**：检查变更间是否有冲突
   - 暂停 A 但另一变更依赖 A → 标注冲突并建议调整顺序
   - 同一 PF 上 modify + pause → 提示矛盾
4. **合并方案**：所有变更的调整方案合并为一个清单
5. **一次确认**：用户确认后批量执行
6. **单次 git commit**：所有变更在一个 commit 中完成
   - commit message 格式：`change(*): batch [N] changes: [摘要]`

## 降级

batch 在降级模式下不可用（无 .devpace/ 无法管理多 CR）。提示用户逐个处理或先初始化。
