# 按变更类型分析与执行细节

> **职责**：核心 5 子命令（add/pause/resume/reprioritize/modify）各自的分析逻辑和执行细节。与 `change-procedures-common.md` 配合加载。

## Step 1 按变更类型分析

**add（插入新需求）**：
- 新需求属于哪个现有功能？需要新建功能分组吗？
- 需要几个 CR？
- 当前迭代容量够吗？需要延后哪些现有功能？

**pause（暂停/砍掉）**：
- 目标功能下有哪些 CR？各自什么状态？
- 有哪些其他 CR 依赖它们？（遍历所有关联关系：阻塞→直接影响、前置→时序调整、关联→通知参考）
- 暂停后对成效指标（MoS）有影响吗？

**resume（恢复已暂停）**：
- 暂停时是什么状态？（读取 CR 的 `暂停前状态` 字段）
- 读取 CR 事件表中最近的 `[checkpoint: ...]` 标记，对比暂停后的代码变更（`git log --since="<暂停日期>"`），判断已通过的门禁是否仍然有效
- 暂停期间代码库是否有变化？质量检查是否仍然有效？
- 恢复后对当前迭代容量的影响？

**reprioritize（优先级调整）**：
- 哪个功能/CR 需要提前？
- 当前正在进行的工作要不要暂停让路？

**modify（修改已有需求）**：
- 已有 CR 的哪些工作需要返工？
- **精确判断检查重置范围**（OPT-05）：
  - 读取 `.devpace/rules/checks.md` 中检查项的 `敏感范围` 字段
  - 对比变更涉及的文件/目录与检查项敏感范围 glob 模式
  - **有交集** → 该检查项需要重置
  - **无交集** → 该检查项保留现有状态
  - **无敏感范围字段** → 保守策略，功能性变更均重置
  - **非功能性变更**（仅文案/注释调整）→ 允许跳过重置，Claude 需说明理由
- CR 意图 section 的哪些字段需要更新？
- **PF 验收标准更新**：
  - 已溢出（`features/PF-xxx.md` 存在）→ 直接更新 PF 文件的验收标准 + 追加 history 注释
  - 未溢出 → 更新 project.md 功能规格 section + 追加 history 注释 + 检查是否触发溢出条件
  - history 注释格式：`<!-- history: [日期] [变更摘要] via /pace-change -->`

## Step 3 按变更类型执行

### CR 文件更新细节

- **pause**：增加 `暂停前状态` 字段，状态改为 paused
- **resume**：移除 `暂停前状态` 字段，状态恢复为暂停前值
- **modify**：更新意图 section，精确重置受影响的质量检查 checkbox 为 `[ ]`（基于敏感范围匹配结果）
- **modify** 事件表备注列记录验收标准差异摘要：`"验收条件 N→M 项: +[新增内容]"`

### PF 文件更新（modify 时）

- 更新 `features/PF-xxx.md` 验收标准 + 追加 history 注释
- 更新 PF 状态（如适用）
- PF 未溢出 → 更新 project.md 功能规格 + 检查溢出条件（本次 modify 可能触发溢出）

## 协同自动化

> OPT-09：add 后容量超出和 pause 释放容量时的协同处理。

**add 后容量超出**：
1. 检测迭代容量（iterations/current.md PF 数 vs 历史速度）
2. 容量超出 → 直接询问用户："迭代容量已满，是否同时调整迭代范围？"
3. 用户确认 → 内联执行 `/pace-plan adjust` 的逻辑（读取 adjust-procedures.md），无需用户手动切换命令
4. 用户拒绝 → 仅记录容量告警到迭代变更记录

**pause 释放容量**：
1. 暂停后计算释放的容量
2. 检查待做池（backlog/ 中 created 状态 CR 关联的 PF）
3. 有待做 PF → 建议："暂停释放了容量，是否把「[PF名]」拉入当前迭代？"

## 下游引导

按变更类型精确引导后续操作：

| 变更类型 | 结果摘要（1-3 行） | 下游引导 |
|---------|------------------|---------|
| add | 新增了什么、影响了哪些 | "开始开发？"→ 确认则进入 /pace-dev |
| modify | 修改了什么、N 项检查需重验 | "N 项检查需重验，继续推进？"→ 确认则恢复 /pace-dev |
| pause | 暂停了什么、释放了容量 | "需要调整迭代范围？"→ 确认则进入 /pace-plan adjust |
| resume | 恢复了什么、质量检查状态 | "继续推进？"→ 确认则进入 /pace-dev |
| reprioritize | 调整了什么 | "切换开始做？"→ 确认则进入 /pace-dev |
| batch | 批量变更摘要 | 综合上述，选择最相关的 1 条引导 |
