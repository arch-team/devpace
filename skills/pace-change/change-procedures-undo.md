# undo 变更撤销规程

> **职责**：undo 子命令的完整流程。自包含，按 SKILL.md 路由表按需加载。

> OPT-10：基于 /pace-change 的 git commit 精确回滚。

## 触发

- 显式：`/pace-change undo`
- 自然语言："撤销刚才的变更""回退上次操作"

## 流程

1. **查找上次变更 commit**：搜索 git log 中最近的 `/pace-change` commit（commit message 含 `change(` 前缀）
2. **展示将回滚的内容**：列出 commit 修改的文件和关键变更
3. **等待确认**：用户确认后执行回滚
4. **执行回滚**：`git revert <commit-hash> --no-edit`
5. **追加记录**：在迭代变更记录表追加"撤销"条目
   - 格式：`| [日期] | undo | 撤销：[原变更描述] | [回滚影响] |`
6. **输出确认**："已撤销变更：[描述]。相关文件已恢复。"

## 限制

- 仅限当前会话内撤销（避免跨会话的状态不一致）
- 如果 revert 产生冲突 → 输出冲突文件列表，提示用户手动处理
- 连续 undo 不支持（撤销的撤销）——建议用户直接重新执行变更
