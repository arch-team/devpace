# monitor — 实时风险汇总

> **职责**：/pace-guard monitor 子命令的详细执行规则。依赖 `guard-procedures-common.md`（分级自主矩阵、分层输出约定）。

## 功能说明

汇总当前 CR 的实时风险状态：读取 CR 文件"运行时风险"section + `.devpace/risks/` 中关联当前 CR 的风险文件。

## 分层输出（A2）

monitor 支持三层输出，默认简要模式减少开发中的信息噪声：

### 简要层（默认）

1 行摘要，适合高频调用（开发中无新增风险时）：

```
CR-003 风险：0 新增 / 1 待处理(M) / 2 已缓解 — 风险可控
```

格式：`{{CR编号}} 风险：{{新增数}} 新增 / {{待处理数}} 待处理({{最高等级}}) / {{已缓解数}} 已缓解 — {{结论}}`

### 标准层（自动升级）

有 Medium/High 新增时自动从简要升级到标准：

```markdown
## 风险监控：{{CR-NNN}}

### 运行时检测
| 时间 | 信号类型 | 等级 | 发现 |
|------|---------|------|------|

### 关联风险
| 编号 | 严重度 | 状态 | 发现摘要 |
|------|--------|------|---------|

**建议**：[基于当前风险状态的行动建议]
```

### 详细层（`--detail` 显式指定）

标准层 + 以下扩展：
- 历史对比：与上次 monitor 输出对比，标注新增/移除/状态变更的风险
- 缓解建议展开：对每个 open 风险附带具体缓解措施建议
- 风险演变时间线：按时间排列风险状态变更历史

### 自动升级规则

| 条件 | 输出层级 |
|------|---------|
| 无新增风险 + 无 Medium/High 待处理 | 简要 |
| 有 Medium/High 新增 | 标准（自动升级） |
| 用户指定 `--detail` | 详细 |
| 用户指定 `--brief` | 强制简要（即使有 Medium/High） |
