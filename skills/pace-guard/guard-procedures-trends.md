# trends — 跨 CR 趋势分析

> **职责**：/pace-guard trends 子命令的详细执行规则。**自包含**——不依赖 guard-procedures-common.md（只读分析，不需要自主权矩阵）。

## 数据采集来源

1. `.devpace/risks/` 所有风险文件（读取来源、严重度、状态、关联 CR、创建时间）
2. `insights.md` 中 defense 类型条目（置信度 + 引用次数）
3. `.devpace/iterations/` 历史迭代文件（关联 CR 列表）

## 趋势计算规则

- **趋势判定**：连续 2 迭代同类型风险增加 = ↑ 恶化；连续 2 迭代减少 = ↓ 改善；其他 = → 稳定
- **重复 Pattern**：按风险类型聚合，取出现频次 Top 3
- **类型分类**：按来源（pre-flight / runtime / retrospective / external）和信号类型分组

## 风险老化检测（B3）

open 状态超过 2 个迭代的风险自动标注老化警告：

1. 计算每个 open 风险的"存活迭代数"：从创建日期到当前迭代的跨度
2. 存活 ≥ 2 迭代 → 在输出中标注 `⏰ 老化`
3. 老化风险自动附带建议：
   - `"RISK-003 已 open N 个迭代，建议评估是否需要升级严重性或标记 resolved。"`
4. 老化风险在趋势报告中单独分组展示（排在重复 Pattern 之后）

## 分层输出（A6）

### 简要层（默认）

3-5 行趋势摘要，适合日常查看：

```
风险趋势：↓ 改善（本迭代新增 2 / 解决 5）
关键模式：依赖影响占比最高(40%)，连续 2 迭代下降
老化风险：1 个（RISK-003，3 迭代未处理）
```

### 详细层（`--detail`）

完整趋势报告：

```markdown
## 风险趋势

### 跨迭代模式（最近 3 迭代）
| 风险类型 | {{Iter-N-2}} | {{Iter-N-1}} | {{Iter-N}} | 趋势 |
|---------|------------|------------|----------|------|

### 重复 Pattern Top 3
1. "{{pattern 描述}}" → 建议：{{行动}}

### 老化风险
| 编号 | 严重度 | 存活迭代数 | 建议 |
|------|--------|-----------|------|

### 风险登记簿状态
- Open: {{N}}（High: {{H}}, Medium: {{M}}, Low: {{L}}）
- 本迭代解决: {{R}}
```

### 自动升级规则

| 条件 | 输出层级 |
|------|---------|
| 日常调用 | 简要（默认） |
| 用户指定 `--detail` | 详细 |
| 被 `/pace-retro` 消费 | 详细（自动升级） |

## 渐进暴露

`.devpace/risks/` 不存在时静默跳过——输出"暂无风险数据。"
