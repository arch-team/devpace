# 迁移框架规程

> **职责**：版本检测和增量迁移规则。从 init-procedures-core.md §8 提取。

## §0 速查卡片

```
版本检测：state.md 末尾 <!-- devpace-version: X.Y.Z --> 标记
无标记 → 视为 v1.2.0 → 触发迁移
标记版本 < 当前版本 → 触发增量迁移
标记版本 = 当前版本 → 不迁移
安全规则：只添加不删除 | 默认值兼容 | 迁移前自动 commit | 幂等
当前迁移链：v1.2.0 → v1.5.0 → v1.6.0
```

## §1 版本检测

检查 `state.md` 末尾的 `<!-- devpace-version: X.Y.Z -->` 标记：

- 无标记 → 视为 v1.2.0（标记引入前的版本），触发迁移
- 标记版本 < 当前版本 → 触发增量迁移
- 标记版本 = 当前版本 → 不迁移

## §2 增量迁移规范

每次版本升级在本文件追加 `vOLD → vNEW` 增量迁移段。迁移前提示用户确认，完成后输出变更摘要。

**通用迁移安全规则**：

- **只添加不删除**：迁移不删除任何现有文件或字段
- **默认值兼容**：新字段全部可选，不填写时使用默认值
- **回滚方案**：迁移前自动 `git commit`（如有未提交变更），迁移后可通过 `git revert` 回滚
- **幂等性**：同一迁移重复执行不产生副作用

## §3 v1.2.0 → v1.5.0 迁移

**触发条件**：`devpace-version` 为 1.2.0（或缺失标记）。

**迁移步骤**：

1. 通知用户："检测到 v1.2.0 项目，建议升级到 v1.5.0 以支持外部同步、增强的质量门和生命周期感知。"
2. 更新版本标记：`<!-- devpace-version: 1.5.0 -->`
3. 更新 rules/workflow.md（从 Plugin 模板覆盖，新增同步相关状态）
4. 输出摘要："升级完成 v1.2.0 → v1.5.0。变更：版本标记更新、工作流规则更新。现有数据无损。"

## §4 v1.5.0 → v1.6.0 迁移

**触发条件**：`devpace-version` 为 1.5.0 或 1.5.1。

**迁移步骤**：

1. 通知用户："检测到 v1.5.x 项目，建议升级到 v1.6.0 以支持信号驱动衔接、CR 级关联和偏好角色。"
2. 更新版本标记：`<!-- devpace-version: 1.6.0 -->`
3. 若 project.md 存在且价值功能树中 PF 行缺少 CR 关联（无 `→ CR-xxx` 格式），在价值功能树 section 末尾追加注释：`<!-- v1.6.0: CR 级关联已支持，新 CR 创建时自动关联到 PF -->`
4. 若 state.md 无 `<!-- taught: ... -->` 标记行，追加：`<!-- taught: -->`（空标记，兼容新版信号驱动教学机制）
5. 输出摘要："升级完成 v1.5.x → v1.6.0。变更：版本标记更新、CR 级关联支持、信号驱动教学标记。现有数据无损。"
