# detail 输出规程

> **职责**：/pace-next detail 模式的输出格式规程。由 SKILL.md 路由表指定加载。

## 输出格式（≤ 8 行）

```
💡 建议：[最高优先级行动]
   原因：[推理]——[推理后缀]
   操作：[命令]（[行为预览]）

📋 候选（按优先级）：
  2. [次优先行动] —— [简述]（[角色]关注）
  3. [第三行动] —— [简述]
```

## 规则

- 遍历全部信号，收集所有命中信号，输出 top-3
- 只有 1 条命中时不展示候选段
- 角色重排序影响的候选标注 `（[角色]关注）`
- 多 developing CR 时，每个 CR 作为独立候选条目，含 PF 上下文
- 命令引导和行为预览见 `next-procedures-output-default.md`
