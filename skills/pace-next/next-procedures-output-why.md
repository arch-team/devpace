# why 输出规程

> **职责**：/pace-next why 模式的输出格式规程。由 SKILL.md 路由表指定加载。

## 输出格式（2-5 行推理链）

用户追问"为什么""怎么判断的"或输入 `/pace-next why` 时，展开推理链：

```
🔍 推理过程：
  1. 信号扫描：扫描了 [N] 个数据源，命中 [M] 个信号
  2. 最高信号：[信号名]（[组名]组）——[条件满足详情]
  3. 优先级对比：[次优信号] 也命中但优先级较低（[原因]）
  4. 角色影响：[角色名] 视角下 [信号X] 提升至 [位置]（无角色影响时省略）
  5. 备选方案：[第二建议概要]
```

## 规则

- 推理链仅在用户追问时触发（被动），不在默认输出中出现
- 行数自适应（2-5 行）：无角色影响或无备选时减少行数
- 命令引导和行为预览见 `next-procedures-output-default.md`
