# 迭代健康度详细规程

> **职责**：迭代健康度评估的详细处理流程（Step 5）。

## Step 5：迭代健康度（health 模式）

1. 读取 `.devpace/iterations/current.md`（不存在则提示"无活跃迭代"）
2. 计算健康度指标：
   - **完成率 vs 时间进度**：PF 完成率 / 迭代时间消耗比（如有周期）；无周期时仅报告 PF 完成率
   - **范围变更次数**：current.md 变更记录条数（反映范围稳定性）
   - **PF 状态分布**：✅ 完成 / 🔄 进行中 / ⏳ 待开始 / ⏸️ 暂停 各多少
   - **迭代速度趋势**：对比上一迭代速度（如有 dashboard.md 数据）
3. 输出健康度摘要：
   - 健康 → "迭代进展正常：完成率 X%，按计划推进"
   - 偏慢 → "迭代可能延期：完成率 X% 但时间已过 Y%，建议 `/pace-plan adjust` 调整范围"
   - 范围不稳定 → "范围变更 N 次，建议关注需求稳定性"
4. **与 pace-pulse 集成**：健康度低于阈值（完成率/时间进度 < 0.5）时，标记为 pulse 可检测信号
