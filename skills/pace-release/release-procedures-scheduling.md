# 发布管理执行规程——发布节奏规划

> **职责**：发布节奏配置、发布窗口提醒和度量。
>
> **按需加载**：由 wizard（发布窗口提醒）或 pace-pulse 按需加载。不对应用户子命令。

## §0 速查卡片

- **节奏配置**：读取 integrations/config.md 的"发布节奏"段（频率/发布日/窗口时间）
- **窗口提醒**：距发布日≤2天→提醒准备 · 已过窗口→提醒 · 刚完成→不提醒
- **联动**：pace-pulse 信号源（Low 级） + pace-plan 迭代结束提示

## 节奏配置

读取 `integrations/config.md` 的"发布节奏"段（如有）：

```markdown
## 发布节奏

- **频率**：[每周/每两周/每月/自定义]
- **发布日**：[周几 或 每月几号]
- **窗口时间**：[如 10:00-16:00]
```

无配置时发布为被动模式（现有行为不变），不提示节奏相关信息。

## 发布窗口提醒

当配置了发布节奏时：

1. **向导增强**：空参数向导在引导结果前检查发布窗口
   - 距发布日 ≤ 2 天 → 提示"📅 发布窗口即将到来（{发布日}），有 N 个 merged CR 可发布。建议准备 Release。"
   - 已过发布日但未发布 → 提示"📅 本周期发布窗口已过（{发布日}），有 N 个 merged CR 待发布。"
   - 刚完成发布（距上次 close ≤ 1 天）→ 不提示

2. **pace-pulse 联动**：当 pace-pulse 执行节奏检测时，发布窗口作为一个信号源
   - 发布窗口到达 + 有 merged CR → 发出"Release 就绪"信号
   - 信号级别：Low（信息性提醒，不暂停工作）

3. **pace-plan 迭代联动**：迭代结束视为自然发布窗口
   - 迭代 close 时如有未发布的 merged CR → 提示"迭代结束，建议运行 /pace-release create 发布本迭代成果。"

## 节奏度量

status history 可展示发布节奏遵循度：
```
发布节奏：每两周（周三）
- 最近 4 个窗口：✅ 准时 · ✅ 准时 · ⚠️ 延迟 1 天 · ❌ 跳过
- 节奏遵循率：75%
```
