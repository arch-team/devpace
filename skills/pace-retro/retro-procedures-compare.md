# compare 对比回顾规程

> **职责**：compare 子命令的完整流程。对比当前迭代与上一迭代的关键指标变化。自包含，按 SKILL.md 路由表按需加载。

## §0 速查卡片

- **触发**：`/pace-retro compare`
- **数据源**：`.devpace/metrics/dashboard.md` 的"度量趋势"表
- **前提**：需要至少 2 个迭代的数据（度量趋势表 ≥2 列）
- **输出**：并排对比表 + delta + 趋势箭头 + 改善/恶化分析
- **只读操作**：不更新任何文件

## 流程

### Step 1：读取历史数据

1. 读取 `.devpace/metrics/dashboard.md`
2. 定位"度量趋势"表（`## 度量趋势`）
3. 解析表头获取迭代列名——最右列为当前迭代（不含"趋势"列），左侧一列为上一迭代
4. 提取两个迭代的所有指标值

### Step 2：数据充分性检查

| 条件 | 行为 |
|------|------|
| dashboard.md 不存在 | 输出："度量仪表盘尚未建立。请先运行 `/pace-retro` 生成基准数据。" |
| 度量趋势表不存在或为空 | 输出："度量趋势表无数据。请先运行 `/pace-retro` 建立基准。" |
| 仅有 1 个迭代数据 | 输出："当前仅有 1 个迭代的度量数据（[迭代名]），对比需要至少 2 个迭代。请在下一次迭代回顾后再使用 compare。" |
| 所有值均为占位符 `—` | 输出："度量数据全部为占位符，请先运行 `/pace-retro` 采集实际数据。" |

### Step 3：计算 Delta 与趋势

对每个指标计算：

1. **Delta**：当前值 - 上一迭代值
   - 百分比指标：直接作差（如 80% → 85%，delta = +5%）
   - 天数指标：直接作差（如 3.0 天 → 1.5 天，delta = -1.5 天）
   - 计数指标：直接作差（如 3 PF → 5 PF，delta = +2 PF）

2. **趋势方向**：
   - 改善类（一次通过率↑、MoS达成↑、迭代速度↑、部署频率↑）：值增大 = ↑ 改善
   - 改善类（打回率↓、CR周期↓、MTTR↓、变更失败率↓）：值减小 = ↑ 改善
   - 变化幅度 <10%（相对值）= → 持平
   - 一方为占位符 `—` = — 不可比

3. **改善/恶化判定**：
   - 改善幅度 >20% = "显著改善"
   - 改善幅度 10-20% = "改善"
   - 变化幅度 <10% = "持平"
   - 恶化幅度 10-20% = "恶化"
   - 恶化幅度 >20% = "显著恶化"

### Step 4：生成对比报告

## 输出格式

```
## 迭代对比：[上一迭代名] → [当前迭代名]

### 指标对比

| 指标 | [上一迭代] | [当前迭代] | Delta | 趋势 | 判定 |
|------|-----------|-----------|-------|------|------|
| 一次通过率 | 70% | 85% | +15% | ↑ | 显著改善 |
| 打回率 | 15% | 8% | -7% | ↑ | 改善 |
| 平均 CR 周期 | 3.0 天 | 1.5 天 | -1.5 天 | ↑ | 显著改善 |
| 迭代速度 | 3 PF | 5 PF | +2 PF | ↑ | 显著改善 |
| MoS 达成率 | 40% | 60% | +20% | ↑ | 显著改善 |
| ...（dashboard.md 度量趋势表中的所有指标）|

### 改善亮点

1. **[指标名]**：[上次值] → [本次值]（[判定]）——[可能原因推测]
2. ...

### 恶化关注

1. **[指标名]**：[上次值] → [本次值]（[判定]）——[建议关注方向]
2. ...

### 综合评估

改善指标 N 项 / 持平 M 项 / 恶化 K 项
整体趋势：[改善 / 持平 / 恶化]（基于改善项数 vs 恶化项数）

→ 深入分析：/pace-retro focus <维度> | 趋势总览：/pace-retro history
```

### 条件段

- **改善亮点**：仅列出判定为"改善"或"显著改善"的指标，无则省略此段
- **恶化关注**：仅列出判定为"恶化"或"显著恶化"的指标，无则省略此段
- **DORA 对比行**：仅当 dashboard.md 有 DORA section 时包含 DORA 指标

### 原因推测规则

对改善/恶化的指标，尝试关联可能原因：
- 读取 `iterations/current.md` 变更记录表，检查是否有相关变更事件
- 读取上次回顾的迭代传递清单（如有），检查建议是否被执行
- 无法归因时标注"原因待分析"

## 规则

- **纯只读**：不更新 dashboard.md、不写入任何文件
- **不执行经验沉淀**：不提炼 pattern
- **不生成迭代传递清单**：不写入 iterations/current.md
- **指标对齐**：对比表的指标行必须与 dashboard.md 度量趋势表的行一一对应，不遗漏不新增
- **占位符处理**：某指标在任一迭代为 `—` 时，该行 Delta 标注"—"，判定标注"不可比"
