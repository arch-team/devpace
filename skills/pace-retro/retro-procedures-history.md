# history 趋势总览规程

> **职责**：history 子命令的完整流程。跨 3+ 迭代的趋势线分析，识别稳定性、拐点和改善斜率。自包含，按 SKILL.md 路由表按需加载。

## §0 速查卡片

- **触发**：`/pace-retro history`
- **数据源**：`.devpace/metrics/dashboard.md` 的"度量趋势"表（全部历史列）
- **前提**：需要至少 3 个迭代的数据（度量趋势表 ≥3 列）
- **输出**：趋势表 + 稳定性分析 + 拐点检测 + 改善斜率
- **只读操作**：不更新任何文件

## 流程

### Step 1：读取全量历史数据

1. 读取 `.devpace/metrics/dashboard.md`
2. 定位"度量趋势"表（`## 度量趋势`）
3. 解析全部迭代列（按时间从左到右排列）和所有指标行
4. 提取每个指标在各迭代的值，构建时间序列

### Step 2：数据充分性检查

| 条件 | 行为 |
|------|------|
| dashboard.md 不存在 | 输出："度量仪表盘尚未建立。请先运行 `/pace-retro` 生成基准数据。" |
| 度量趋势表不存在或为空 | 输出："度量趋势表无数据。请先运行 `/pace-retro` 建立基准。" |
| 仅有 1-2 个迭代数据 | 输出："当前仅有 [N] 个迭代的度量数据，趋势分析需要至少 3 个迭代。当前可用：`/pace-retro compare`（双迭代对比）。" |

### Step 3：统计分析

对每个指标的时间序列计算：

1. **描述统计**：
   - 均值（Mean）
   - 最大值 / 最小值
   - 标准差（SD）——衡量波动程度
   - 变异系数（CV = SD / Mean）——标准化波动率

2. **稳定性判定**（基于变异系数）：

   | CV 范围 | 判定 | 说明 |
   |---------|------|------|
   | CV < 0.05 | 高度稳定 | 几乎无波动 |
   | 0.05 ≤ CV < 0.15 | 稳定 | 正常范围波动 |
   | 0.15 ≤ CV < 0.30 | 中等波动 | 需关注成因 |
   | CV ≥ 0.30 | 高波动 | 过程不稳定，需干预 |

3. **拐点检测**：识别趋势方向发生逆转的迭代
   - 拐点定义：连续 ≥2 个迭代同方向变化后，方向逆转
   - 记录拐点位置（迭代名）和方向（改善→恶化 或 恶化→改善）

4. **改善斜率**：
   - 使用最近 3 个数据点计算简单线性趋势
   - 斜率为正（指标含义为"越大越好"时）= 改善趋势
   - 斜率为负（指标含义为"越小越好"时）= 改善趋势
   - 斜率方向与指标改善方向一致时标注 ↑，反向标注 ↓，接近 0 标注 →

### Step 4：生成趋势报告

## 输出格式

```
## 趋势总览（[最早迭代] → [最新迭代]，共 [N] 个迭代）

### 指标趋势

| 指标 | [Iter-1] | [Iter-2] | ... | [Iter-N] | 均值 | 波动 | 趋势 |
|------|----------|----------|-----|----------|------|------|------|
| 一次通过率 | 70% | 80% | ... | 90% | 80% | 稳定 | ↑ 持续改善 |
| 平均 CR 周期 | 3.0 天 | 2.5 天 | ... | 1.5 天 | 2.3 天 | 稳定 | ↑ 持续改善 |

### 稳定性分析

**高度稳定**（CV < 0.05）：
- [指标列表]——过程成熟，无需额外关注

**高波动**（CV ≥ 0.30）：
- **[指标名]**（CV = X.XX）——[可能原因：迭代间范围差异大/流程不稳定/外部因素]

### 拐点检测

| 指标 | 拐点迭代 | 方向 | 前序趋势 | 后序趋势 |
|------|---------|------|---------|---------|
| [指标名] | [迭代名] | 恶化→改善 | ↓ 连续 N 迭代 | ↑ 连续 M 迭代 |

（无拐点时输出："所有指标趋势方向一致，未检测到拐点。"）

### 改善斜率

| 指标 | 近 3 迭代斜率 | 改善速率 | 预测 |
|------|-------------|---------|------|
| [指标名] | [+/-X.X/迭代] | 快/中/慢/停滞 | 按当前速率，[N] 个迭代后可达 [目标] |

**改善速率判定**：
- 快：每迭代改善 >15%
- 中：每迭代改善 5%-15%
- 慢：每迭代改善 <5%
- 停滞：变化 <1% 或方向不一致

### 综合评估

**最佳表现指标**：[指标名]——连续 [N] 迭代改善，当前 [分级] 级
**最需关注指标**：[指标名]——[波动大/持续恶化/停滞不前]，建议 [改进方向]
**整体研发节奏**：[稳定向好 / 波动中改善 / 需要关注]

→ 聚焦分析：/pace-retro focus <维度> | 双迭代对比：/pace-retro compare
```

### 条件段

- **DORA 趋势行**：仅当 dashboard.md 有 DORA section 且有 ≥3 迭代数据时包含
- **知识库趋势行**：仅当 dashboard.md 有知识库 section 时包含
- **拐点检测段**：无拐点时输出单行提示
- **预测列**：仅当指标有明确目标值（如 MoS 目标、DORA Elite 标准）时提供预测

## 规则

- **纯只读**：不更新 dashboard.md、不写入任何文件
- **不执行经验沉淀**：不提炼 pattern
- **不生成迭代传递清单**：不写入 iterations/current.md
- **统计精度**：百分比保留整数位，天数保留 1 位小数，CV 保留 2 位小数
- **占位符处理**：某指标在某迭代为 `—` 时，该数据点排除在统计之外，有效数据点 <3 时该指标标注"数据不足"
- **表格宽度控制**：迭代列 >6 时，仅显示最近 6 列 + 最早 1 列（中间用 `...` 省略）
