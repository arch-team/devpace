# mid 中期检查规程

> **职责**：mid 子命令的完整流程。迭代中期的轻量版回顾，快速检查进度和早期预警。自包含，按 SKILL.md 路由表按需加载。

## §0 速查卡片

- **触发**：`/pace-retro mid`
- **定位**：迭代进行中的快速检查点，不是正式回顾
- **数据范围**：只采集"进行中"和"已完成"的 CR 数据
- **输出**：进度 snapshot + 质量信号 + 早期预警
- **不更新 dashboard.md**：避免中期数据污染基准线
- **不执行经验沉淀**：Step 4 跳过
- **不生成迭代传递清单**：非迭代末尾

## 流程

### Step 1：采集当前进度数据

仅采集当前状态快照，不进行历史对比。

**产品功能进度**（从 `iterations/current.md`）：
- 各 PF 状态分布：⏳ 待开始 / 🔄 进行中 / ✅ 完成 / ⏸️ 暂停
- PF 完成率 = ✅ 数 / 总 PF 数
- 进行中 PF 的 CR 完成进度

**CR 状态分布**（从 `.devpace/backlog/`）：
- 筛选当前迭代关联的 CR（通过 iterations/current.md 产品功能表的 CR 列表）
- 各 CR 状态：created / developing / verifying / in_review / approved / merged / released / paused
- 仅统计"进行中"（developing/in_review）和"已完成"（approved/merged）的 CR

**质量信号**（从已完成 CR 的事件表）：
- 已完成 CR 的质量检查一次通过率
- 已完成 CR 的人类打回次数（in_review→developing 事件数）
- 打回率 = 打回次数 / 总 review 次数

### Step 2：阈值检查与早期预警

对采集数据执行阈值检查，识别需要关注的信号：

| 检查项 | 预警阈值 | 预警级别 | 建议动作 |
|--------|---------|---------|---------|
| PF 完成率 | 迭代过半但完成率 <30% | **红色** | 评估范围，考虑缩减 PF |
| PF 完成率 | 迭代过半但完成率 <50% | **黄色** | 关注进度，优先完成高优先级 PF |
| 一次通过率 | <60% | **红色** | 质量问题，需审视开发流程 |
| 一次通过率 | <75% | **黄色** | 质量信号偏低，关注常见问题 |
| 打回率 | >25% | **红色** | 频繁打回，review 标准或实现质量需关注 |
| 打回率 | >15% | **黄色** | 打回偏多，建议分析打回原因 |
| paused CR 数 | >总 CR 数的 30% | **黄色** | 阻塞较多，需排查阻塞原因 |
| 无进展 PF | 状态仍为 ⏳ 且迭代过半 | **黄色** | 未启动的 PF 需重新评估优先级 |

**迭代进度判定**：
- 读取 `iterations/current.md` 的周期字段（起止日期）
- 计算已过天数 / 总天数 = 时间进度百分比
- 无明确结束日期时跳过时间相关的预警检查

### Step 3：生成中期检查报告

## 输出格式

```
## 中期检查：[迭代名称]（[时间进度]%）

### 进度 Snapshot

📊 PF 完成率：[M/N]（[X%]）| 时间进度：[Y%]
| 状态 | PF 数 | 占比 |
|------|-------|------|
| ✅ 完成 | [N] | [X%] |
| 🔄 进行中 | [N] | [X%] |
| ⏳ 待开始 | [N] | [X%] |
| ⏸️ 暂停 | [N] | [X%] |

CR 状态：merged [N] · in_review [N] · developing [N] · created [N] · paused [N]

### 质量信号

| 指标 | 当前值 | 状态 |
|------|--------|------|
| 一次通过率（已完成 CR） | [X%] | [🟢/🟡/🔴] |
| 打回率（已完成 CR） | [Y%] | [🟢/🟡/🔴] |
| 样本量 | [N 个已完成 CR] | — |

### 早期预警

[🔴/🟡 预警项列表——按预警级别排序，红色优先]
1. 🔴 **[检查项]**：[当前值]（阈值 [X]）→ 建议：[动作]
2. 🟡 **[检查项]**：[当前值]（阈值 [X]）→ 建议：[动作]

（无预警时输出："🟢 所有检查项在正常范围内。"）

### 进行中 PF 概览

| PF | 标题 | CR 进度 | 预估完成度 |
|----|------|--------|-----------|
| [PF 编号] | [标题] | [merged/总数] | [基于 CR 进度的完成度估算] |

→ 聚焦某维度：/pace-retro focus <维度> | 完整回顾：/pace-retro
```

### 条件段

- **早期预警段**：无预警时用单行绿色提示替代
- **质量信号段**：无已完成 CR 时标注"尚无已完成 CR，质量信号待积累"
- **时间进度**：无明确周期时标注"（周期未设定）"

## 规则

- **不更新 dashboard.md**：中期数据是过程快照，不应写入基准线，避免污染趋势数据
- **不执行经验沉淀**：不提炼 pattern、不调用 pace-learn
- **不生成迭代传递清单**：不写入 iterations/current.md
- **不输出报告质量自评**：中期检查不需要
- **样本量透明**：质量信号必须标注样本量（N 个已完成 CR），让用户了解数据代表性
- **预警而非判定**：预警是提醒信号，不是最终结论。用"建议"而非"必须"措辞
- **轻量优先**：总输出控制在一屏以内（约 30-40 行），突出关键信号
