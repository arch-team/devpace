# update 执行规程

> **职责**：`/pace-retro update` 的专用规则。完成 Step 1（数据收集）→ Step 2（dashboard 更新）后输出变化反馈，然后结束。

## §0 速查卡片

- **触发**：`/pace-retro update`
- **流程**：Step 1 数据收集（优先使用脚本，见下方）→ Step 2 更新 dashboard.md → 输出变化反馈
- **脚本采集**：`Bash: node ${CLAUDE_PLUGIN_ROOT}/scripts/compute-metrics.mjs .devpace`——输出 JSON 含 8 核心指标，直接用于 Step 2 dashboard 更新。脚本不可用时降级为共享规程（`retro-procedures-common.md`）
- **不生成报告**：不执行 Step 3-6
- **历史快照**：更新时追加到"度量趋势"表，不覆盖旧值

## 历史快照管理

每次更新 dashboard.md 时，不直接覆盖旧值，而是追加到"度量趋势"表：

```markdown
## 度量趋势

| 指标 | {{Iter-1}} | {{Iter-2}} | {{Iter-N}} | 趋势 |
|------|-----------|-----------|-----------|------|
| 一次通过率 | 70% | 80% | 85% | ↑↑ 持续改善 |
| 平均 CR 周期 | 3.0 天 | 2.5 天 | 1.5 天 | ↑↑ 显著改善 |
| 打回率 | 15% | 10% | 8% | ↑↑ 持续改善 |
| 迭代速度 | 3 PF | 4 PF | 5 PF | ↑↑ 持续提升 |
```

**趋势判定规则**：
- 连续 3+ 迭代单调改善 = "↑↑ 持续改善"
- 连续 2 迭代改善 = "↑ 改善中"
- 波动（先升后降或反复）= "〰️ 波动"
- 连续 2+ 迭代恶化 = "↓ 需关注"
- 稳定（变化幅度 <10%）= "→ 稳定"

**首次度量**：趋势表仅含当前迭代一列，趋势标注"— 首次"。

## update 模式反馈

`/pace-retro update` 完成 Step 1→2 后，输出变化反馈摘要：

```
✅ dashboard.md 已更新（N 项变化）：
- [指标名]：[旧值] → [新值] ↑/↓
- [指标名]：[旧值] → [新值] ↑/↓
- ...

无变化的指标已保持原值。
```

**规则**：
- 只列出有变化的指标（变化幅度 ≥1% 或绝对值变化 ≥0.1）
- 标注趋势方向箭头
- 若无任何变化，输出"✅ dashboard.md 已检查，所有指标无变化。"
- 首次度量时输出"✅ dashboard.md 已建立基准线（N 项指标）。"
