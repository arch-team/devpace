# since：时间维度修饰符

> 由 SKILL.md 路由表加载。仅在 `since` 修饰符时读取。

`since <时间标记>` 作为修饰符与其他子命令组合使用，不增加选项总数。

## 时间标记解析

| 格式 | 含义 | 示例 |
|------|------|------|
| `Nd` | 最近 N 天 | `since 3d` = 最近 3 天 |
| `Nw` | 最近 N 周 | `since 1w` = 最近 1 周 |
| `last-session` | 上次会话结束以来 | 读取 state.md 最近更新时间 |

## 数据采集

1. **Git 历史**：`git log --since="<时间>" --name-only -- .devpace/` 获取文件变更列表
2. **CR 事件表**：扫描 backlog/ 中 CR 事件表，筛选时间范围内的事件
3. **迭代变更**：读取 iterations/ 变更记录中时间范围内的条目

## 输出格式（独立使用）

```
📅 最近 3 天变更摘要
新增：[CR 标题 A]（创建 → 开发中）
完成：[CR 标题 B]（merged）
状态变更：[CR 标题 C]（开发中 → 等待审批）
```

## 与子命令组合

- `/pace-status since 3d` → 最近 3 天变更摘要（独立使用）
- `/pace-status detail since 3d` → detail 输出中高亮最近 3 天有变化的功能
- 其他子命令 + since → 在子命令输出中标注时间范围内的变更

## 导航

独立使用时末尾追加 1 行：`→ 查看当前全貌：/pace-status detail`

与其他子命令组合时，跟随该子命令的导航。
