# link / create — 关联与创建

> **职责**：定义 /pace-sync link 和 create 子命令的详细执行步骤。

## §1 link 输入解析

`$1` = CR-ID（如 CR-003 或 003），`$2` = 外部 ID（如 #42 或 42，**可选**）

## §2 标准 link（提供外部 ID）

**执行步骤**：
1. 验证 CR 存在：检查 `.devpace/backlog/CR-{id}.md`
2. 验证外部实体存在：执行适配器"获取状态"操作（参数：外部实体编号）
3. 写入 CR 文件"外部关联"字段（格式由适配器定义，含完整 URL）
4. 更新 sync-mapping.md 关联记录表（追加行）
5. 输出确认：CR-{id} ↔ 外部实体 #{编号} 已关联

## §3 智能 link（省略外部 ID）

当用户仅提供 CR 编号时，自动搜索外部平台中标题相似的实体：

**执行步骤**：
1. 验证 CR 存在，读取 CR 标题
2. 执行适配器"列出工作项"操作，获取外部平台的开放实体列表
3. 按标题相似度排序，取前 5 个候选
4. 展示候选列表供用户选择：
   ```
   找到以下可能匹配的 Issue：
   1. #42 — 实现用户认证模块 (open, backlog)
   2. #38 — 用户登录功能 (open, in-progress)
   3. #55 — 认证系统重构 (open)
   选择编号关联（或输入外部 ID）：
   ```
5. 用户选择后执行 §2 标准 link 流程

**无匹配时**：提示"未找到匹配的外部实体。可用 /pace-sync create {CR-ID} 创建新 Issue。"

## §4 批量 link

`$ARGUMENTS` 包含 `--all` 时，为所有未关联的 CR 逐个执行智能 link（§3）。

**执行步骤**：
1. 读取 sync-mapping.md 关联记录，识别已关联 CR
2. 扫描 `.devpace/backlog/` 中所有 CR，筛选未关联的
3. 对每个未关联 CR 执行 §3 智能 link
4. 输出汇总：`{N} 个 CR 已关联 / {M} 个跳过（用户取消或无匹配）`

## §5 关联记录格式

（按 Plugin `knowledge/_schema/sync-mapping-format.md`）：
```markdown
| CR-{id} | {平台}#{编号} | {YYYY-MM-DD HH:mm} | — |
```

## §5.1 错误处理

- CR 不存在 → 提示用户
- 外部实体不存在 → 提示用户确认 ID
- CR 已有关联 → 提示已关联，确认是否覆盖

## §6 create — 从 CR 创建外部工作项

**输入**：`$1` = CR-ID

**执行步骤**：
1. 验证 CR 存在且无外部关联
2. 读取 CR 元数据：标题、意图描述、验收条件、关联 PF
3. 生成工作项描述：
   ```
   ## 变更请求：{CR 标题}

   **意图**：{意图描述}

   **验收条件**：
   {逐条列出验收条件}

   **关联功能**：{PF 名称}

   ---
   _由 devpace 自动创建_
   ```
4. 查询状态映射表获取当前状态对应的外部状态标记
5. 执行适配器"创建工作项"操作（参数：标题、描述、状态标记）
6. 自动执行 link（复用 §2 流程）
7. 输出确认：CR-{id} → 外部工作项 #{编号} 已创建并关联

**错误处理**：
- CR 已有外部关联 → 提示已关联，确认是否创建新工作项并覆盖
- 平台工具不可用 → 提示安装
- CR 不存在 → 提示用户
