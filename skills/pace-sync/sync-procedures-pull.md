# pull — 轻量外部状态检查

> **职责**：查询外部状态并提示用户是否更新 devpace 状态。不自动修改内部状态。支持全部实体类型（EPIC/BR/PF/CR）。

## 输入

`$1` = 实体 ID（必填，通过 `sync-procedures-entity.md` §1 解析）

## 执行步骤

1. 解析实体 ID（调用 entity.md §1）→ 确定实体类型和文件位置
2. 验证实体存在且有外部关联（独立文件查"外部关联"字段；内联实体查 sync-mapping.md 关联记录）
3. 读取实体当前 devpace 状态
4. 通过操作语义"获取实体类型状态映射"确定对应的状态映射表
5. 执行适配器"获取状态"操作 → 查询外部当前状态
6. 比较 devpace 状态与外部状态的映射一致性
7. **一致** → 输出"{实体ID} 与外部状态一致（{devpace 状态} ↔ {外部状态}）"
8. **不一致** → 输出差异并提示：
   ```
   {实体ID} 状态不一致：
   - devpace: {devpace 状态}
   - 外部 (#{编号}): {外部状态}（映射到 devpace: {推断状态}）

   是否更新 devpace 状态为 {推断状态}？
   注意：Epic/BR/PF 状态为聚合计算值，通常不应从外部反向修改。CR 状态转换需符合状态机规则。
   [更新 / 跳过]
   ```
9. 用户确认更新 →
   - **CR**：验证状态转换合法性（参照状态机规则）→ 合法则更新，不合法则提示走正常工作流
   - **Epic/BR/PF**：警告"此实体状态由下游实体聚合计算，直接修改可能导致不一致。建议通过修改下游 CR 状态来间接更新。确认强制修改？"

**降级**：平台工具不可用 → 提示安装。实体无外部关联 → 提示先 link 或运行 /pace-sync。
