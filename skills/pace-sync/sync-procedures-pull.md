# pull — 轻量外部状态检查（MVP）

> **职责**：查询外部状态并提示用户是否更新 devpace 状态。不自动修改内部状态（因为 devpace 状态机有转换规则）。

## 输入

`$1` = CR-ID（必填）

## 执行步骤

1. 验证 CR 存在且有外部关联
2. 读取 CR 当前 devpace 状态
3. 执行适配器"获取状态"操作 → 查询外部当前状态
4. 比较 devpace 状态与外部状态的映射一致性
5. **一致** → 输出"CR-{id} 与外部状态一致（{devpace 状态} ↔ {外部状态}）"
6. **不一致** → 输出差异并提示：
   ```
   CR-{id} 状态不一致：
   - devpace: {devpace 状态}
   - 外部 (#{编号}): {外部状态}（映射到 devpace: {推断状态}）

   是否更新 devpace 状态为 {推断状态}？
   注意：此转换需要符合 devpace 状态机规则。
   [更新 / 跳过]
   ```
7. 用户确认更新 → 验证状态转换是否合法（参照 `rules/devpace-rules.md` §2 状态转换规则）
   - 合法 → 更新 CR 状态，输出确认
   - 不合法 → 输出"状态转换 {当前} → {目标} 不合法（{原因}），建议通过正常工作流推进"

**降级**：平台工具不可用 → 提示安装。CR 无外部关联 → 提示先 link。
