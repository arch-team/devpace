# status / unlink — 状态查看与解除关联

> **职责**：定义 /pace-sync status 和 unlink 子命令的执行步骤。支持全部实体类型。

## §1 status — 同步状态查看

**执行步骤**：
1. 读取 sync-mapping.md 关联记录
2. **模式判断**：`$ARGUMENTS` 包含 `--all` 时扫描全部实体类型；否则仅显示 CR（向后兼容）
3. 对每个关联记录：
   a. 读取实体当前状态
   b. 执行适配器"获取状态"操作（可选——平台工具不可用时显示"未知"）
   c. 比较一致性
4. 输出同步状态表

## §2 status 输出格式

**默认（仅 CR）**：
```
| CR | 外部链接 | devpace 状态 | 外部状态 | 一致性 | 最后同步 |
|----|---------|-------------|---------|--------|---------|
| CR-003 | #42 | developing | in-progress | ✅ | 02-25 10:30 |
| CR-005 | #18 | merged | open | ❌ 需同步 | 02-24 15:00 |
```

**--all 模式（全实体）**：
```
| 实体 | 类型 | 外部链接 | devpace 状态 | 外部状态 | 一致性 | 最后同步 |
|------|------|---------|-------------|---------|--------|---------|
| EPIC-001 | Epic | [#10](URL) | 进行中 | in-progress | ✅ | 03-25 10:30 |
| BR-001 | BR | [#11](URL) | 进行中 | in-progress | ✅ | 03-25 10:30 |
| CR-003 | CR | [#42](URL) | developing | in-progress | ✅ | 03-25 10:30 |
| CR-005 | CR | [#18](URL) | merged | open | ❌ 需同步 | 03-24 15:00 |

汇总：3 一致 / 1 需同步 / 4 未关联
提示：运行 /pace-sync 执行智能同步
```

## §3 无关联记录时

```
当前没有已关联的外部实体。
运行 /pace-sync 开始智能同步（自动检测并创建/推送）。
或用 /pace-sync link {实体ID} #{Issue编号} 手动关联。
```

## §4 unlink — 解除关联

**输入**：`$1` = 实体 ID（通过 `sync-procedures-entity.md` §1 解析）

**执行步骤**：
1. 验证实体存在且有外部关联
2. 清除实体文件中的"外部关联"字段（独立文件）；内联实体仅更新 sync-mapping.md
3. 从 sync-mapping.md 关联记录表中移除对应行
4. 输出确认：{实体ID} 已解除与 {外部实体} 的关联

## §5 unlink 错误处理

- 实体无外部关联 → 提示"{实体ID} 当前没有外部关联"
- 实体不存在 → 提示用户
