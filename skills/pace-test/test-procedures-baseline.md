# 测试执行规程——测试基准线（baseline）

> **职责**：/pace-test baseline 测试基准线——建立/更新基准、趋势对比。自包含。

## §0 速查卡片

- **测试基准线**：建立/更新测试基准（通过率+耗时），支持趋势对比
- **数据来源**：执行 `test-procedures-core.md` §1 获取当前测试结果
- **持久化**：写入 `.devpace/rules/test-baseline.md`（格式遵循 `knowledge/_schema/test-baseline-format.md`）

## 流程

当用户执行 `/pace-test baseline` 时，建立或更新测试执行基准线，供 /pace-retro 度量使用。

1. **执行完整测试**：运行 `test-procedures-core.md` §1 基础测试执行流程，获取当前测试结果
2. **收集基准数据**：
   - 每项检查的通过状态和执行耗时
   - 总通过率
   - 总执行时间
   - 如果 `test-strategy.md` 存在 → 需求覆盖率
3. **对比历史基准**（如存在）：
   - 读取 `.devpace/rules/test-baseline.md`
   - 对比当前结果与上一次基准：
     - 通过率变化（↑/↓/→）
     - 耗时变化（快/慢/持平）
     - 新增/删除的检查项
4. **写入基准文件**：生成或更新 `.devpace/rules/test-baseline.md`

### 基准文件格式

格式遵循 `knowledge/_schema/test-baseline-format.md`（当前基准表 + 检查项明细表 + 历史趋势表）。

## 输出格式

```markdown
## 测试基准线更新

**状态**：[新建基准 / 更新基准]

### 当前基准

- 检查项：N 项（通过 X / 失败 Y）
- 通过率：Z%
- 执行时间：W.Ws

### 与上次基准对比（如有）

| 指标 | 上次 | 本次 | 变化 |
|------|------|------|------|
| 通过率 | X% | Y% | ↑ +N% |
| 执行时间 | As | Bs | ↓ 快 Cs |
| 检查项数 | M | N | +K 新增 |

基准文件已更新：`.devpace/rules/test-baseline.md`
```

### --brief 输出格式

`基准：通过率 X% · 耗时 Ys · 趋势 [↑/↓/→]`

## 降级场景

| 条件 | 行为 |
|------|------|
| 无 checks.md 且无可检测测试 | 输出"无可执行的测试命令，无法建立基准" |
| 首次建立基准 | 不做对比，直接生成初始基准文件 |
| 测试全部失败 | 仍记录基准（基准反映实际状态，非理想状态）|
