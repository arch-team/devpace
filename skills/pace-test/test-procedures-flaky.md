# 测试执行规程——不稳定测试分析（flaky）

> **职责**：/pace-test flaky 不稳定测试分析与主动维护检测。自包含。

## §0 速查卡片

- **不稳定测试分析**：收集执行历史→识别间歇/环境/顺序/超时四类模式→主动维护检测
- **测试健康度分析**：空断言/执行膨胀/长期未更新/死测试的主动扫描
- **闭环**：不稳定测试→降级 test-strategy.md 对应条目状态

## 流程

当用户执行 `/pace-test flaky` 时，进行**不稳定测试分析**和**主动测试维护检测**——从响应式维护升级为主动式维护。

1. **收集测试执行历史**：
   - 扫描 `.devpace/backlog/` 中所有 CR 文件的事件表 → 提取以下类型的测试记录：
     - `/pace-test 执行` 类型的事件
     - Gate 1/Gate 2 事件中包含 "passed"/"failed" 关键词的检查结果（通过 /pace-dev 执行的测试）
   - 从 `insights.md`（如存在）提取测试相关的 pattern（`category: 质量保障`）
   - 从 `git log` 提取最近 N 次测试相关的提交（包含 test/fix 关键词）
2. **识别不稳定模式**：
   - **间歇失败**：同一检查项在多次执行中状态不一致（有时 PASS 有时 FAIL）
   - **环境依赖**：失败集中在特定时间段或特定 CR 上下文
   - **顺序依赖**：仅在特定执行顺序下失败
   - **超时波动**：同一检查项耗时方差超过 50%
3. **主动测试维护检测**（在不稳定分析之后执行）：
   - **永远通过的测试**：扫描测试文件，检测可能的空断言模式——无 assert/expect 语句、仅 `pass`/`return true` 的测试函数、被注释掉的断言
   - **执行时间膨胀**：对比 test-baseline.md（如存在）的历史耗时数据，标记耗时增长 >50% 的检查项
   - **长期未更新的测试**：通过 `git log` 检查测试文件最后修改时间，标记 >30 次 CR 未修改的测试文件（可能与源代码已脱节）
   - **死测试**：扫描被 `skip`/`xfail`/`@Disabled`/`.skip` 标记的测试，统计数量和分布
4. **评估影响**：
   - 不稳定测试导致的 Gate 1 误拦截次数（从 CR 事件表的重试记录推断）
   - 受影响的 PF 范围
5. **生成建议**
6. **结果持久化**（通过 pace-learn 统一写入管道）：
   - 将发现的不稳定测试 pattern 和维护问题构造为学习请求：
     ```
     请求来源：pace-test flaky
     事件类型：flaky-detection
     建议类型：防御
     建议标签：质量保障
     描述：[模式描述]
     证据：[受影响 CR 列表、执行历史]
     ```
   - 交给 pace-learn Step 3 统一写入管道处理（去重 + 置信度初始化 + 格式合规）
   - 无发现时不生成学习请求
7. **回写 test-strategy.md**（flaky→strategy 闭环）：
   - 如果 `.devpace/rules/test-strategy.md` 存在且 Step 2 识别到不稳定测试：
     - 将不稳定测试对应的验收条件状态从 `✅ 已有` 降级为 `⚠️ 不稳定`
     - 更新策略总览表的"已覆盖"计数（不稳定测试不计入有效覆盖）
     - 在降级条目的"测试文件/位置"列追加注释：`（不稳定：[模式]）`
   - test-strategy.md 不存在 → 跳过
   - **与 accept Step 5.5 的关系**：accept 检测弱覆盖/虚假覆盖 → 降级；flaky 检测不稳定 → 降级。两者独立触发，共同维护 strategy 数据准确性

## 输出格式

```markdown
## 测试健康度分析

**分析范围**：最近 N 个 CR 的测试执行记录

### 不稳定测试

| # | 检查项 | 失败率 | 模式 | 影响 CR |
|---|--------|--------|------|---------|
| 1 | [检查名] | 2/5 (40%) | 间歇失败 | CR-003, CR-005 |

### 测试维护问题

| # | 问题类型 | 测试文件/检查项 | 详情 | 建议 |
|---|---------|---------------|------|------|
| 1 | 空断言 | tests/test_utils.py::test_helper | 无 assert 语句 | 补充断言或删除 |
| 2 | 耗时膨胀 | lint 检查 | 1.2s → 3.8s (+217%) | 检查新增文件量或优化配置 |
| 3 | 长期未更新 | tests/test_auth.py | 最后修改 CR-002（30+ CR 前） | 审查是否与源代码脱节 |
| 4 | 已跳过 | tests/test_legacy.py::test_old | @skip 标记 | 修复或删除 |

### 建议

| # | 检查项/文件 | 建议操作 | 理由 |
|---|------------|---------|------|
| 1 | [检查名] | 修复：[具体方向] | [原因分析] |
| 2 | [检查名] | 隔离：移至 Gate 2 意图检查 | 无法自动化稳定验证 |

### 修复优先级

1. **立即修复**：影响 P0 功能的不稳定测试 + 空断言测试（虚假安全感）
2. **计划修复**：间歇失败但有明确根因 + 耗时膨胀（CI 瓶颈）
3. **可暂时隔离**：低优先级或环境相关的不稳定测试
4. **清理建议**：长期跳过的测试 + 长期未更新的测试
```

### --brief 输出格式

`健康度：不稳定 N · 维护问题 M · 优先修复 K`

## 降级场景

| 条件 | 行为 |
|------|------|
| 无 CR 事件表记录 | 不稳定分析输出"无历史测试数据"，但主动维护检测仍执行（扫描测试文件不需要历史数据） |
| 仅 1 次执行记录 | 不稳定分析输出"数据不足"，主动维护检测仍执行 |
| 无不稳定测试且无维护问题 | 输出"测试健康度良好 ✅" |
| 无测试文件 | 输出"未检测到测试文件，建议先执行 /pace-test generate 生成测试框架" |
