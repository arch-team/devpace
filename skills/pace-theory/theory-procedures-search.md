# theory-procedures-search — 关键词搜索输出规则

> 由 SKILL.md 路由表加载。仅 `<关键词>` 子命令时读取。自包含，不依赖其他 procedures。

## §0 速查卡片

```
搜索流程：Grep theory.md → 匹配段落 + 上下文定位 + 子命令推荐
无匹配时 fallback 推荐 2-3 个最接近的子命令
```

## §1 搜索流程

1. 用 Grep 在 `knowledge/theory.md` 中搜索用户关键词
2. 匹配结果处理：
   - **无匹配**："未找到'[关键词]'的匹配内容。相关主题：[推荐 2-3 个最接近的子命令]"
   - **单匹配**：输出匹配段落（前后 3 行上下文）+ 定位信息
   - **多匹配（≤5）**：输出匹配数量 + 最相关 2-3 段 + 各自定位
   - **多匹配（>5）**：输出匹配数量 + 最相关 3 段 + "结果较多，建议用具体子命令查询"

## §2 上下文定位格式

每个搜索结果附带定位：

```
📍 此概念属于 [章节名]（§N）
   相关子命令：/pace-theory [子命令1]、[子命令2]
```
