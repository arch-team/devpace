## 输出格式框架（三段叙事）

结论先行 → 评估过程 → 决策上下文，符合审计用户"为什么 → 怎么判的 → 基于什么"的思维顺序。

````markdown
## 决策轨迹 — [CR 名称] [决策类型]

### 判定结论
[最终判定] ——[关键因素]
重建置信度：[高/中/低]（[置信度依据]）

### 评估过程

#### [类型特定标题]
[Gate 类型：逐项检查列表 / 分析类型：分析维度列表]

#### 规则匹配
- [匹配的规则/检查项]: [如何应用到当前场景]

#### 经验影响（如有）
- [pattern 名称]（置信度 [N]）→ 影响了 [具体判断]

### 决策上下文

#### 读取的信息
- [文件路径]: [读取了什么信息]

#### 溯源信息
- 用户明确表达: [source: user 的内容摘要]
- Claude 推断: [source: claude 的内容摘要及推断原因]

→ [上下文感知的导航建议]
````

**类型差异**：Gate 类型的"评估过程"以"逐项检查"为标题，含编号检查项列表；分析类型的标题和内容由各类型文件定义（见 `trace-procedures-analysis.md`）。

## 重建置信度（confidence）

基于数据完整性的启发式评估——反映"轨迹重建的可靠程度"，而非"决策本身的正确性"。

| 数据完整性 | 置信度 | 条件 |
|-----------|--------|------|
| 完整 | 高 | 事件表 + 溯源标记 + checkpoint 证据 + insights 引用记录全部存在 |
| 较完整 | 中 | 事件表 + checkpoint 证据存在，溯源标记或 insights 引用部分缺失 |
| 部分 | 低 | 仅事件表存在，checkpoint 证据和溯源标记均缺失 |

置信度展示在"判定结论"section 中，括号内说明依据。

## 降级处理

按场景分级输出，引导用户自助解决。

| 场景 | 检测条件 | 输出 | 引导 |
|------|---------|------|------|
| 项目未初始化 | `.devpace/` 目录不存在 | "未找到 .devpace/ 目录，无法查询决策轨迹。" | → /pace-init 初始化项目 |
| CR 未找到 | backlog/ 中无匹配 CR | "未找到匹配 '[input]' 的变更请求。" | → /pace-status detail 查看所有变更请求 |
| 决策类型无数据 | CR 存在但无对应类型的 checkpoint 或事件 | "该变更请求无 [type] 类型的决策记录。" | 列出该 CR 可用的决策类型（从事件表提取） |
| 溯源标记缺失 | CR 有事件记录但无 `<!-- source: -->` 标记 | "有事件记录但缺少溯源标记，轨迹重建不完整。" | "推进模式中自动生成溯源标记，当前轨迹基于事件表重建" |
| 未知类型 | 输入类型不在支持列表中 | "未知决策类型 '[input]'。" | 列出支持的类型：gate1/gate2/gate3/intent/change/risk/autonomy/timeline |

## 执行步骤

1. **解析参数**：识别 CR（名称匹配或 ID）和决策类型
2. **降级检查**：按降级处理表依次检测，命中任一场景则输出对应提示并终止
3. **定位数据源**：
   - CR 文件（`backlog/CR-*.md`）：事件表、意图 section、质量检查 checkbox
   - project.md：溯源标记、功能树上下文、自主级别
   - checks.md：项目质量规则
   - insights.md：经验引用记录（查找 `[insight-ref: ...]` 备注）
4. **评估数据完整性**：根据 confidence 评估表判定置信度
5. **重建决策上下文**：根据事件表时间线，还原决策时 Claude 读取了哪些信息
6. **展示溯源标记**：提取并展示 CR 意图 section 和 project.md 中的 `<!-- source: ... -->` 标记
7. **提取经验影响**：从事件表备注列查找 `[insight-ref: ...]` 标记，展示经验对决策的影响
8. **输出格式化轨迹**：按三段叙事格式输出（具体格式差异见类型文件）
9. **附加导航建议**：根据决策类型附加上下文导航（导航表见类型文件）

## 规则

- **只读操作**：不修改任何文件（纯查询）
- **基于已有数据**：从 CR 事件表、checkpoint 标记、溯源标记重建轨迹，不重新执行判断
- **溯源标记展示**：深入层是溯源标记唯一可见的位置（日常输出不展示）
- **结论先行**：判定结论放在最前面，满足审计用户"先看结果"的需求
- **confidence 必须展示**：每次输出都包含重建置信度评估
- **降级分级**：不同失败场景输出不同提示和引导，不用单一降级消息
- **导航不遗漏**：每次成功输出后都附加导航建议
- **不暴露内部术语**：输出中使用自然语言描述，不直接展示 schema 字段名
