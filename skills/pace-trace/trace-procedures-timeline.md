## 输出格式

从 CR 事件表提取所有 checkpoint 标记和状态转换，按时间线呈现全生命周期决策总览。

````markdown
## 决策时间线 — [CR 名称]

| 时间 | 决策点 | 结果 | 关键因素 |
|------|--------|------|---------|
| [日期] | 意图检查 | [结果摘要] | [关键因素] |
| [日期] | Gate 1 | [✅ 通过 / ❌ 未通过] | [证据摘要] |
| [日期] | Gate 2 | [✅ 通过 / ❌ 未通过] | [证据摘要] |
| [日期] | Gate 3 | [✅ 通过 / ❌ 打回] | [原因] |
| ... | ... | ... | ... |

重建置信度：[高/中/低]（[基于事件表完整性]）

→ 某节点完整轨迹：/pace-trace [CR] [类型]
→ CR 当前状态：/pace-status detail
````

## 数据提取规则

1. 读取 CR 文件事件表的所有行
2. 提取含 `[checkpoint:` 标记的行 → 解析类型和证据
3. 提取所有状态转换事件（`→` 分隔的状态变化）
4. 按时间排序，合并为时间线表格
5. "关键因素"列从事件表"备注"列提取，checkpoint 证据优先

## 执行步骤

1. **解析参数**：识别 CR（名称匹配或 ID）
2. **降级检查**：按降级处理表依次检测，命中则输出提示并终止
3. **扫描事件表**：读取 CR 文件事件表全部行，提取 checkpoint 标记和状态转换
4. **构建时间线**：按时间排序合并为表格
5. **输出格式化**：使用 timeline 专用格式
6. **附加导航**：`→ 某节点完整轨迹：/pace-trace [CR] [类型] · 当前状态：/pace-status detail`

## 降级处理

| 场景 | 检测条件 | 输出 | 引导 |
|------|---------|------|------|
| 项目未初始化 | `.devpace/` 目录不存在 | "未找到 .devpace/ 目录，无法查询决策轨迹。" | → /pace-init 初始化项目 |
| CR 未找到 | backlog/ 中无匹配 CR | "未找到匹配 '[input]' 的变更请求。" | → /pace-status detail 查看所有变更请求 |
| 决策类型无数据 | CR 存在但无 checkpoint 或状态转换事件 | "该变更请求无决策记录。" | 列出该 CR 可用的决策类型（从事件表提取） |

## 规则

- **只读操作**：不修改任何文件（纯查询）
- **基于已有数据**：从事件表重建时间线，不重新执行判断
- **不暴露内部术语**：输出中使用自然语言描述，不直接展示 schema 字段名
