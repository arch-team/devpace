# 场景 S02: 日常推进

**对应需求**: S2, F1.2, F1.3, F1.6
**验证目标**: 上下文自动恢复、CR 状态推进、intent checkpoint

## 前置条件

- 已完成 S01（.devpace/ 已初始化）
- 至少存在一个产品功能待实现

## 执行步骤

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 开始新会话 | 自动读取 state.md，1 句话报告当前状态 |
| 2 | 说"继续" | 从上次断点恢复，进入推进模式 |
| 3 | 说"帮我做 [功能名]" | 自动创建 CR（不暴露 CR-xxx），进入 developing |
| 4 | 等待 Claude 工作 | 每个原子步骤后 git commit + 更新 CR + 更新 state.md |
| 5 | 会话结束 | 保存进度到 state.md，输出 3-5 行摘要 |
| 6 | 重新开始会话 | 从 state.md 恢复，报告上次进度 |

## 验收标准

- [ ] 会话开始自动读取 state.md
- [ ] 用 1 句自然语言报告状态（不暴露 ID 和状态机术语）
- [ ] 用户说"继续"能从上次断点恢复
- [ ] 自动创建 CR 并关联到匹配的产品功能
- [ ] CR 状态正确流转（created → developing → verifying）
- [ ] 意图检查点在首次进入 developing 时执行

## 评分维度

| 维度 | Pass | Partial | Fail |
|------|------|---------|------|
| 上下文恢复 | 准确恢复到上次断点 | 恢复但遗漏部分状态 | 无法恢复 |
| 自然语言 | 零 ID/术语暴露 | 偶尔暴露 ID | 大量技术术语 |
| 状态流转 | 完整且正确 | 跳过状态但功能正常 | 状态错误 |
| Checkpoint | 每步保存 | 偶尔遗漏保存 | 不保存 |

## 检查产物

- `.devpace/state.md` — 更新后的当前工作和下一步
- `.devpace/backlog/CR-*.md` — 状态、事件表、意图 section
- `git log` — 原子步骤提交记录
