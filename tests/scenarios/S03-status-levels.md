# 场景 S03: 查看进度

**对应需求**: S3, F1.5, NF6, NF9
**验证目标**: 三级输出粒度（1 行/detail/tree）

## 前置条件

- 已完成 S01 + S02（有进行中和已完成的工作）

## 执行步骤

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 输入 `/pace-status` | 输出 1-3 行概览 |
| 2 | 输入 `/pace-status detail` | 展开到产品功能级别 |
| 3 | 输入 `/pace-status tree` | 展示完整价值功能树 |
| 4 | 输入 `/pace-status metrics` | 展示度量仪表盘 |
| 5 | 输入 `/pace-status [关键词]` | 搜索匹配的 CR 详情 |

## 验收标准

- [ ] 默认输出 ≤ 3 行概览
- [ ] detail 模式展开到产品功能级别（PF 名称 + 状态）
- [ ] tree 模式展示完整 OBJ → BR → PF → CR 链路
- [ ] 不主动暴露 ID（除非用户明确要求）

## 评分维度

| 维度 | Pass | Partial | Fail |
|------|------|---------|------|
| 输出粒度 | 三级分明 | 粒度区分不清 | 只有一种输出 |
| 信息准确性 | 反映真实状态 | 部分过时 | 信息错误 |
| 自然语言 | 零术语/ID | 偶尔暴露 | 大量技术术语 |
