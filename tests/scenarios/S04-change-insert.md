# 场景 S04: 需求插入

**对应需求**: S4, F2.1-F2.3
**验证目标**: 影响分析、方案等待确认、多文件更新

## 前置条件

- 已完成 S01（.devpace/ 已初始化）
- 至少有 1 个进行中的 CR

## 执行步骤

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 说"还要加一个 [新功能]" | 识别为"插入新需求"变更 |
| 2 | 等待影响分析 | 读取功能树和迭代计划，自然语言报告影响 |
| 3 | 等待调整方案 | 提出方案（新建 PF+CR、排期建议），等待确认 |
| 4 | 说"好的，加上" | 执行变更：创建 PF/CR，更新功能树、迭代计划、state.md |
| 5 | 验证产物 | 所有关联文件已更新 |

## 验收标准

- [ ] 自动识别"新需求"意图
- [ ] 执行影响分析（影响哪些 PF/CR，迭代容量评估）
- [ ] 用自然语言报告影响（不使用 ID 和技术术语）
- [ ] 提出调整方案并等待确认（不自作主张执行）
- [ ] 确认后自动更新所有相关文件

## 评分维度

| 维度 | Pass | Partial | Fail |
|------|------|---------|------|
| 变更识别 | 正确识别变更类型 | 需要二次确认 | 未识别 |
| 影响分析 | 覆盖全部受影响对象 | 遗漏部分 | 无影响分析 |
| 等待确认 | 严格等待 | 部分自动执行 | 全部自动执行 |
| 文件更新 | 全部更新 | 遗漏 1-2 个文件 | 核心文件未更新 |

## 检查产物

- `.devpace/project.md` — 价值功能树新增条目
- `.devpace/backlog/CR-*.md` — 新建的 CR 文件
- `.devpace/iterations/current.md` — 变更记录表新增条目
- `.devpace/state.md` — 反映新增需求
