# 场景 S05: 需求暂停

**对应需求**: S5, F2.4
**验证目标**: paused 状态保留、依赖解除、恢复流程

## 前置条件

- 已完成 S01 + S02
- 至少有 1 个 developing 状态的 CR

## 执行步骤

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 说"[功能名] 先不做了" | 识别为"暂停"变更 |
| 2 | 等待影响分析 | 报告影响范围和依赖关系 |
| 3 | 确认暂停 | CR → paused，功能树标 ⏸️，解除阻塞 |
| 4 | 继续其他工作... | 正常推进不受暂停影响 |
| 5 | 说"恢复 [功能名]" | 从 paused 恢复到暂停前状态 |
| 6 | 验证恢复 | 功能树恢复标记，CR 回到暂停前状态 |

## 验收标准

- [ ] CR 标记为 paused，工作成果保留
- [ ] 依赖此 CR 的其他 CR 自动解除阻塞
- [ ] 功能树用 ⏸️ 标记暂停项
- [ ] 恢复时回到暂停前状态
- [ ] 恢复后代码库有变化时重新验证质量检查

## 评分维度

| 维度 | Pass | Partial | Fail |
|------|------|---------|------|
| 暂停保留 | 分支/代码/检查进度全保留 | 部分丢失 | 工作成果丢失 |
| 依赖处理 | 自动解除阻塞 | 手动提示处理 | 未处理 |
| 恢复流程 | 完整恢复到暂停前 | 恢复但状态不准确 | 无法恢复 |
| 文件同步 | 全部文件更新 | 遗漏部分 | 核心文件未更新 |

## 检查产物

- `.devpace/backlog/CR-*.md` — 状态 paused，含`暂停前状态`字段
- `.devpace/project.md` — 功能树 ⏸️ 标记
- `.devpace/iterations/current.md` — 变更记录表新增暂停/恢复条目
- `.devpace/state.md` — 反映暂停/恢复
