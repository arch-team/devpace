# 场景 V: 新项目端到端验证

**对应需求**: V2.14-V2.16
**验证目标**: 空项目初始化 + 完整 CR 周期 + 跨会话

## 前置条件

- 一个全新的空项目（仅含基础文件如 README.md）
- devpace 插件已加载

## 执行步骤

### Phase 1: 初始化（V2.14）

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 进入空项目目录 | 无 .devpace/，正常工作 |
| 2 | 输入 `/pace-init` | 对话引导初始化 |
| 3 | 提供项目信息 | 生成完整 .devpace/ |
| 4 | 验证初始化 | 所有文件完整，模板已替换 |

### Phase 2: 完整 CR 周期（V2.15）

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 说"帮我做 [第一个功能]" | 自动创建 CR，进入推进模式 |
| 2 | 等待 Claude 完成 | created → developing → verifying |
| 3 | 触发 review | verifying → in_review + 变更摘要 |
| 4 | 说"通过" | approved → merged + 连锁更新 |
| 5 | 验证价值功能树 | CR emoji 更新，PF 状态可能更新 |

### Phase 3: 跨会话连续性（V2.16）

| # | 人类动作 | 期望 Claude 行为 |
|---|---------|-----------------|
| 1 | 开始第二个功能后中断 | checkpoint 保存 |
| 2 | 新会话开始 | 自动恢复，报告进度 |
| 3 | 继续完成 | 从断点恢复工作 |

## 验收标准

- [ ] 空项目 /pace-init 到首个 CR merged 流程完整
- [ ] 价值功能树从零建立并正确更新
- [ ] 跨会话中断后自动恢复，零手动解释
- [ ] 所有 .devpace/ 文件格式正确

## 评分维度

| 维度 | Pass | Partial | Fail |
|------|------|---------|------|
| 初始化 | 一次成功，文件完整 | 需要修正 | 初始化失败 |
| CR 周期 | 完整走通 6 个状态 | 跳过部分状态 | 流程断裂 |
| 跨会话 | 完美恢复 | 恢复但信息不全 | 无法恢复 |
| 整体体验 | 流畅自然 | 需少量指导 | 频繁卡住 |

## 检查产物

- `.devpace/` 全部文件 — 格式正确性
- `git log` — 完整的提交历史
- `.devpace/backlog/CR-*.md` — 至少一个 merged 状态的 CR
- `.devpace/project.md` — 价值功能树含 ✅ 标记
